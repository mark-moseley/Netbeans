Using the Relation Output Operator Property Editor

See Also

The Relation Output operator sends out groups of tuples that reflect its input; when data changes on the current relation result. It sends out records with a tag on the end which indicates  + or -  (records added or deleted, respectively.)

However, if a triggering event enters into the IEP process, but the result of the final Relation from that event does not change,  the relation output is not triggered. 

If a change occurs as a result of the triggering event such that a record changes, a delete record is sent out matching the changed record.  An add record is sent out when there is an updated or new version of that record. The operator only sends out change events rather than the entire result time after time. 
For example, if a change occurs which results in a new record,  only an add record is sent out.  Again, if the new result is that only three records no longer appear,  three delete records are sent out.

The Attribute fields in the Relation Output are supplied by the input that the operator receives.

To create a Relation Output operator.

  1. Drag a Relation Output operator to the Design view.
  2. Click the Relation Output operator icon.
  3. Click the Include Timestamp checkbox if you want to include a timestamp on the output from the Relation Output.
  4. Click Ok.
