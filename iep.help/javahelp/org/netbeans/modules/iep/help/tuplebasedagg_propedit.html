Using the Tuple Based Aggregator Operator Property Window

See Also

The Tuple Based Aggregator performs statistical analysis for a specified number of records, for every increment you specify, and provides the ability to do SQL selects based on the specified records.

Use the Tuple Based Aggregator when you want to perform calculations based on an increment of the data that is input to the operator. For example, provided a stream of stock transactions, the operator can compute a new stream that holds the minimum, average, and maximum of the stock price of every 10 transactions.

Statistics you can compute via SQL statements within the Proeprty Editor of the Tuple Based Aggregator include:

  • Sum
  • Average
  • Median
  • Lower-quantile
  • Upper-quantile
  • Minimum
  • Maximum
  • Standard-deviation
  • Sum-of-square
  • Outlier-percentage

To create a Tuple Based Aggregator operator:

  1. Drag a Tuple Based Aggregation operator from the Palette to the Design area.
  2. Click the Tuple Based Aggregation operator icon.The Property Editor opens.
  3. In the Start field, enter the record in which you want the operator calculations to begin.
  4. In the Increment field, enter how often you want the analysis performed.

To define the SQL statements to use with a Time Based Aggregator operator:

  1. In the Expression box enter the expression for the SQL Select statement which is used to specify the input attribute upon which the Select is performed.
  2. In the From box, define the input on which to perform the selection.
  3. In the Where clause box, provide additional filtering on records, for example, WHERE price > 30.00 AND stockDate < '2006-01-01' ;
  4. In the Group by box, group a result into subsets that have matching values for one or more columns of the database by specifying a comma separated list of qualified attribute names.
