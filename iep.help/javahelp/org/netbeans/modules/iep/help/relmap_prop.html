Using the Relation Map Operator Property Editor

See Also

The Relation Map operator performs a select on two or more incoming relations, equivalent to a SQL join view of a minmum of one relation and additional tables and relations. The Relation Map operator can take multiple inputs.

You can use a Relation Map to join input from other operator sources, for example, two or more Tuple Based windows, or two or more Partitioned Windows.

To create a Relation Map Operator

  1. Drag a Relation Map operator from the Palette to the Design view.
  2. Connect at least two inputs.
  3. Click the Relation Map operator icon to examine the Details and Attribute sections in the Property Editor, if needed.
  4. Click OK.

To define the SQL statements to use with a Relation Map operator:

  1. In the Expression box, enter the expression for the SQL Select statement. Add attribute names, data types and sizes, as applicable.
  2. In the From field, define the input on which to perform the selection.
  3. In the Where clause field, provide additional search criteria.
