IBM WebSphere Application Server Properties

See Also

You access this property sheet by right-clicking an IBM WebSphere application server instance node in the Runtime window and choosing Properties.

Properties

  • Display Name. Specifies the name of your application server for display purposes in the Runtime window. This property is set when you register your application server in the IDE using the Add Server Instance wizard.
  • URL. Shows the full URL address that will be used when you deploy to this server instance. This property is read-only.
  • SOAP Port. Specifies the number of the port for the server SOAP connector. This property is read-only.
  • HTTP Admin Port. Specifies the number of the port to access the administration server. This property is read-only.
  • HTTP Port. Specifies the number of the TCP/IP port that the server listens on. This property is read-only.
  • Username. Specifies the username the IDE will use when starting the server. The value you supplied when you registered the server instance is the default value for this field.
  • Password. Specifies the password required for the user specified in Username. The value you supplied when you registered the server instance is the default value for this field.
  • Server Root. Shows the root directory for the local server instance. This directory is the server's domain root. This property is read-only.
  • Profile Root. Shows the root directory of the profile of the server instance. This property is read-only.
  • Debugger Port. Specifies the number of the TCP/IP port that the IDE connects to when the server is started in debug mode.
