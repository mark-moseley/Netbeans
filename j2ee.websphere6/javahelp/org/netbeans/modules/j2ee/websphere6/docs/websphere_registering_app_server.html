Registering an IBM WebSphere Application Server

See Also

Before you can start developing and deploying applications to an application server, you must register your application server with the IDE.

To register an instance of an IBM WebSphere application server:

  1. Choose Tools > Servers in the main menu.
  2. In the Server Manager, click Add Server.
    • Alternatively, right-click the Servers node in the Services window and choose Add Server from the popup menu.
    The Add Server Instance wizard starts.
  3. In the Choose Server pane, do the following:
    1. Select the version of the IBM WebSphere Application Server that you want to register.
    2. In the Name field, type a name for the server instance.
      This name is used to identify the server in the IDE.
    3. Click Next.
  4. In the Server Location pane, use the Browse button to navigate to and select the installation directory of your application server.
    • The Next button is only enabled when you have selected the correct installation directory.
  5. Click Next.
  6. To define a local server instance, do the following:
    1. In the Instance Properties pane, from the Local/Remote drop-down list, select Local.
    2. From the Local instances drop-down list, select a server instance.
      When you select a server instance, the IDE updates the Profile path, Host, and Port fields.
    3. (Optional) Fill in the Username and Password fields.
  7. To define a remote server instance, do the following:
    1. In the Instance Properties pane, from the Local/Remote drop-down list, select Remote.
    2. Define the location of the server in the Host and Port fields.
    3. (Optional) Fill in the Username and Password fields.
  8. Click Finish.
  9. If you started the Add Server Instance wizard from the Server Manager, click Close in the Server Manager dialog box.
    The IDE displays a server node for your application server in the Services window under the Servers node.
