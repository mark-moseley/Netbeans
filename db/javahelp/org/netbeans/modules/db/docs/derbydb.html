Connecting to a Derby Database

See Also

The IDE includes a bundled Derby database server. You can create and work with Derby databases without installing any additional software.

Version 8.2 of the Sun Java System Application Server also bundles the Derby database server. When you register the application server, the IDE switches from using the bundled Derby installation to using the installation in the application server. The IDE also registers the connections to the application server's sample databases in the Runtime window. All of the databases that you created with the IDE's bundled Derby installation are still available.

To connect to a Derby database:

  1. In the Runtime window, expand Databases > Drivers.
  2. Right-click the appropriate Apache Derby driver node and choose Connect Using.
  3. Enter the URL of the database. The URL format is given in the textbox:
    jdbc:derby://<HOST>[:<PORT>]/<DB>
  4. Enter the User name and Password. By default, the Derby database server has an empty username and password.
  5. Click OK. The IDE registers the connection under the Databases node.

To create a new Derby database:

  1. Choost Tools > Derby Database > Create Derby Database from the main window.
  2. Enter a name for the database.
  3. Optionally, select a new location for the database folder and click OK. The IDE starts the database server if it is not already running, creates the database, and registers the connection under the Databases node.

Notes:

  • Selecting Remember password will store the password for the current IDE session. If left unselected, a dialog box appears prompting you for the password every time you connect to the database.
