Set Quick Filter Dialog Box

See Also

You use the Set Quick Filter dialog box to specify classes or packages that you want to exclude or include when analyzing application performance. Unlike global filters or custom filter sets, a quick filter can be applied only to the current application.

You can access this window by selecting Quick Filter from the filter drop-down list in the Select Profiling Task dialog box.

You can set the following properties in the dialog box:

  • Filter Type. Select Exclusive or Inclusive to specify if the methods are excluded or included. If Exclude is selected, none of the listed methods are profiled. If Include is selected, only the listed methods are profiled.
  • Filter Value. This text area lists the classes and packages that you want to include or exclude. You can type the name in the text area in the following formats:
    java.*
javax.swing.
javax.xml.parsers.SAXParser

    You can quickly add classes in your project to the Quick Filter by doing the following:

    1. Select the Profile Only Project Classes Filter in the Select Profiling Task dialog box.
    2. Click Show Filter Value to open the Profile Only Project Classes dialog box.
    3. Click To Quickfilter.
    When you click To Quickfilter, the Set Quick Filter dialog box opens and the project classes are listed in the Filter Value text box. You can delete or add classes by typing in the text box.
