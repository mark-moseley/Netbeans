Table Layout Dialog Box

See Also

The Table Layout dialog box enables you to specify the layout and contents of a Table component. You open the dialog box by right-clicking a Table component in the Visual Designer and choosing Table Layout.

This dialog box is especially useful if you have added a database table to the current page, as described in Adding Tables to a Page. You can use the Get Data From drop down list in the Columns tab to see a list of appropriate objects that are in scope, such as the data provider for a database table that you previously dropped on the page.

When the dialog box opens, the Columns tab is selected. If you dropped a table directly on the Table component, you see the table's columns and fields in the dialog box, and you see the name of the table's data provider in the Get Data From field. If you did not drop a table on the Table component (for example, you dropped it on the page), you can use the dialog box's Get Data From drop down list to pick the data provider.

The dialog box has the following tabs and fields:

  • Columns tab. Use this tab to determine the object that displays in the table. If you are using a database table, you can determine the columns of the database table that are displayed, the order in which the columns are displayed, and the JavaServer Faces components that are available to the user for viewing or working with the table.
    • Get Data From. You can select any object listed.

      When you select a table data provider from the Get Data From drop down list, the available columns appear in the Available list. However, when you select an array or List, the IDE does not know what kinds of objects will be in that array or List at runtime, so nothing appears in the Available list. Instead, you must use the Value Expression field to specify a JavaServer Faces expression for that column. The expression should be relative to the currentRow (begins with #{currentRow), but the IDE accepts any valid JavaServer Faces expression that can be resolved at design time. You can add new columns to the table with the New button.

    • Available. Shows all the columns in the database table. Select a column if you want to add it to the list of displayed columns.
    • Selected. Shows the columns to be displayed in the Table component. Select a column if you want to delete it or move it.
      • Use the <, >, and << buttons to add columns to and remove columns from this list.
      • Use the Up and Down buttons to change the display order of the columns.
      • Use the New button to add a column, such as a special column that is not a database table column.
    • Column Details. Shows properties of the column selected in the Displayed list.
      • Header Text. Text displayed at the top this column. For advanced users: You can insert a JavaServer Faces expression in the header.
      • Footer Text. Text displayed at the bottom of this column. For advanced users: You can insert a JavaServer Faces expression in the footer.
      • Component Type. The type of JavaServer Faces component used in the column. You can choose any component from the drop down list to display it in the selected column of the table. The same component will display in every cell of the column. Another way to display a component in a column of a table is to drag the component from the Palette and drop it on the column. If you drop a component on a column, you might want to delete the component already in the column, which by default is an Output Text component.
      • Value Expression. A value expression used to get the value for the column. This field is not enabled when the table is bound to a database, but it does show the value expression used with the database. If the table is bound to an array, you use the value expression to get the value to be used for the column. The expression should be relative to the currentRow (begins with #{currentRow), but the IDE accepts any valid JavaServer Faces expression that can be resolved at design time.
      • Width. The width of the column in pixels.
      • Horizontal Align. Alignment of the column contents relative to the left and right sides of the cell. Equivalent to the <td> tag's align parameter.
      • Vertical Align. Alignment of the column contents relative to the top and bottom of the cell. Equivalent to the <td> tag's valign parameter.
      • Sortable. Indicates whether or not the user can sort the column. If selected, this property causes a sort button  to appear on the right side of the column header.
  • Options tab. Use this tab to set various display options for the table, such as its title, header, and footer text, buttons that do things like selecting and deselecting all rows, and determining if the table is paginated.
    • Title. Sets the table's title property, which determines the title displayed at the top of the table.
    • Summary. Sets the table's summary property, which provides information about the Table's purpose or structure or both that is readable by a screen reader.
    • Footer. Sets the table's footerText property, which determines the footer text displayed at the bottom of the table.
    • Empty Data Message. The message displayed if the table is submitted without data.
    • Show Select All Rows Button. Sets the table's selectMultipleButton property, which displays a button that the user can click to check an entire column of checkboxes, selecting all rows in the table.
    • Show Deselect All Rows Button. Sets the table's deselectMultipleButton property, which displays a button that the user can click to uncheck an entire column of checkboxes, deselecting all rows in the table.
    • Show Clear Sort Button. Sets the table's clearSortButton property, which displays a Clear Table Sort button that the user can click to remove all sorting currently applied to the table, returning the table items to the order they were in when the page was initially rendered.
    • Show Sort Panel Toggle Button. Sets the table's sortPanelToggleButton property, which displays a button that the user can click to open and close the table's Sort panel.
    • Enable Pagination. Makes it possible to display the table as a paginated table, useful for tables with many rows. Sets the following properties:
      • paginateButton - Displays a button that enables users to switch between viewing all data on a single page, an unpaginated table, and seeing data in multiple pages, a paginated table.
      • paginationControls - Displays table pagination controls that enable users to change which page is displayed. The controls include an input field for specifying the page number, a Go button to go to the specified page, and buttons for going to the first, last, previous, and next page
    • Page Size. Determines the number of rows that display on a paginated page.
