Importing a Java Studio Creator Project

See Also | Tutorials

You can import an existing project from Java Studio Creator 2 , Java Studio Creator 2 Update 1, or NetBeans Visual Web Pack 5.5 for use in NetBeans 6.0. If the project uses a data source, the IDE configures a data connection and shows it under the Drivers node in the   You can simplify connecting to a Java DB database by installing and registering an instance of the Sun JavaTM System Application Server 9.1 on your system.   • If you do not have the target server set correctly, the IDE prompts you to do so. Right-click the project in the Projects window and choose Resolve Missing Server Problem.
To import a Java Studio Creator 2 or Visual Web Pack project:
  1. In the main toolbar, choose File > Open Project and browse for the project that you want to import.
    • Before you open the existing project, the IDE prompts you to make a backup of your project. When you import and change a Java Studio Creator or Visual Web Pack project, you may no longer use the project in Java Studio Creator 2 or NetBeans Visual Web Pack 5.5. When you have made a copy of the project, click OK to continue.
  2. Click Open Project Folder to open the project in the IDE.

    The IDE scans the project classpath.

  3. Restart the IDE.
  4. In the Services Window, expand the Databases node , and then right-click the node of the database you want to use and choose Connect.

    The IDE prompts you to enter a password for the database.

  5. In the Projects window, expand the project node, right-click the Data Source References node, and choose Refresh.

    If there are unresolved data sources, a red broken badge   is displayed on the Data Source References node.

    • If more than one project is imported that uses the same data source, right-click the Data Source References node and choose Resolve Data Source(s) to add JNDI resources for the Sun Java System Application Server.
  6. If there are unresolved data sources, right-click the Data Source References node and choose Resolve Data Source(s).

    The Data Source References node turns red and the Resolve Data Source(s) dialog box opens.

    • When you choose Refresh or Resolve Data Source(s) from the pop-up menu, it takes time to determine the status of the project's data sources, so the mouse cursor changes to a wait cursor.
  7. In the Resolve Data Source(s) dialog box, select the database name from the list and click Add Data Connection.

    The Add Database Connection dialog box opens.

    • If the project you are importing uses a Java DB database, go to step 14.
  8. To add a new server type, in the Server Type drop-down list, select Add Server Type.

    The Edit Database Server Types dialog box opens

  9. Click New and browse for the database driver JAR file.

    The driver JAR file then appears in the JDBC Driver Jar File(s) pane.

  10. If nothing appears in the Driver Class Name field, click Suggest.

    The IDE suggests the best possible driver class name for the driver.

  11. If nothing appears in the URL Template field, type the URL template for the database connection. Follow the example that is provided.
    • For Java DB databases, the URL should look like this: jdbc:derby://localhost:1527/travel.
  12. Click Close.

    The Add Data Connection dialog box opens.

  13. Select the server type from the drop-down list.
  14. Enter a User ID and a Password.
    • For Java DB databases, the password is the same as the User ID. For example, for the Travel sample database the user name is travel and the password is travel.

  15. Enter the Database URL and click Add to add the Data Source.
  • For a database other than Java DB, if you plan to run the project on Sun JavaTM System Application Server 9.1, you must copy the database driver libraries to the server's lib directory prior to running the project. Also, you must ensure that the port numbers used by the project match the port numbers required by your installation of Sun Application Server 9.1.
