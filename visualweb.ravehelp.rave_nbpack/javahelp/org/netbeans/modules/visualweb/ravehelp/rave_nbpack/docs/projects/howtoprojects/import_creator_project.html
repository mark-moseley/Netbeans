Importing a Legacy Project

See Also | Tutorials

A legacy project is a project that was created in Java Studio Creator 2 , Java Studio Creator 2 Update 1, or NetBeans Visual Web Pack 5.5. You can import a legacy project in NetBeans 6.0. For more information about importing legacy projects, go to

  • Tips for Migrating Java Studio Creator 2 Update 1 Projects to NetBeans
  • Tips for NetBeans Visual Web Pack Applications to NetBeans

    • To import a legacy project:

    1. Import the settings from a previous version of the IDE
    2. In the main toolbar, choose File > Open Project.
    3. In the Open Project dialog box, browse for the project.
    4. Click Open Project Folder to open the project in the IDE.

      The IDE scans the project classpath.

    5. If you are connecting to a Java DB database using the Sun Java System Application Server 9, restart the IDE.

    Connecting to a Database

    In NetBeans 6.0, you must resolve all data sources before you run the application. When you migrate the Java Studio Creator 2.1 user directory, third party complibs migrate to the NetBeans 6.0 directory and appear in the Component Library Manager.

    1. In the Services Window, expand the Servers node, right-click the server you want to use, and choose Start.
    2. After the server has started, expand the Databases node , and then right-click the node of the database you want to use and choose Connect.

      The IDE prompts you to enter a password for the database.

    Resolving Data Sources

    In NetBeans 6.0, you must resolve all data sources before you run the application. When you migrate the Java Studio Creator 2.1 user directory, third party complibs migrate to the NetBeans 6.0 directory and appear in the Component Library Manager.

    1. In the Projects window, the project node is red and displays a badge when there is a missing data source.
    2. To resolve a data source, right-click the Data Source References node and choose Resolve Data Sources.
    3. In the Resolve Data Source(s) dialog box, select the database name from the list and click Add Data Connection.

      The Resolve Data Sources dialog box opens, displaying data sources that do not have a database connection.

    4. Select the data souce you want to resolve and click OK.

      The Add Database Connection dialog box opens.

      • Note: If the project you are importing uses a Java DB database, go to step 9.
    5. To add a new server type, in the Server Type drop down list, select Add Server Type.

      The Edit Database Server Types dialog box opens

    6. Click New and browse for the database driver JAR file.

      The driver JAR file then appears in the JDBC Driver Jar File(s) pane.

    7. If nothing appears in the Driver Class Name field, click Suggest.

      The IDE suggests the best possible driver class name for the driver.

    8. If nothing appears in the URL Template field, type the URL template for the database connection. Follow the example that is provided.
      • Note: For Java DB databases, the URL should look like this: jdbc:derby://localhost:1527/travel.
    9. Click Close.

      The Add Data Connection dialog box opens.

    10. Select the server type from the drop down list.
    11. Enter a User ID and a Password.

    12. Enter the Database URL and click Add to add the Data Source.
    • Note: For a database other than Java DB, if you plan to run the project on Sun JavaTM System Application Server 9.1, you must copy the database driver libraries to the server's lib directory prior to running the project. Also, you must ensure that the port numbers used by the project match the port numbers required by your installation of Sun Application Server 9.1.
