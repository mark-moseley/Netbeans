Faces Verbatim Component

See Also

This advanced component is a custom <f:verbatim/> tag that enables you to add HTML elements to certain components and have the HTML elements act like components . Wrapping an HTML tag in an <f:verbatim/> tag makes it behave as though it were a JavaServer Faces component, solving some problems with pages that have mixtures of JavaServer Faces components and HTML elements.

  • The Markup component is easier to use for adding an HTML tag to your JSP page.

If you put an HTML element or plain text inside a JavaServer Faces component like Table or Grid Panel and you do not use the Faces Verbatim component, the following happens:

  • The HTML component does not appear at all at design time because Table and Grid Panel are set to render their own children. However, HTML elements are not really children of the JavaServer Faces component, so both the Visual Designer and the component ignore the embedded elements.
  • At runtime, the tab order of the embedded HTML elements will be different from that of any embedded JavaServer Faces components. So if the user tries to tab through the components embedded in the JavaServer Faces component, there is no guarantee that the tab order will proceed in an orderly fashion.
  • If you use a Faces Verbatim component, be sure to follow these guidelines:
    • The code in JSP pages used by the IDE must be balanced and well formed XML, including the markup inside an <f:verbatim> tag. For example, you cannot start a <table> in an <f:verbatim> tag but put the ending </table> tag after the ending </f:verbatim> tag.
    • The <f:verbatim> component cannot have other JavaServer Faces components embedded inside it.
