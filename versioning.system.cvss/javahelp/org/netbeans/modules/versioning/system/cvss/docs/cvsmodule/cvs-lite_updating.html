Updating Files in a Local Working Directory

Updating files prior to committing changes made to local copies of files is important to avoid merge conflicts and ensure that any changes made in the remote repository are incorporated locally.

To update a local version of a file:
  1. Right-click the file and choose CVS > Update.

    2. The IDE incorporates any changes existing in the repository version of the file.

Updating Projects with Dependencies

If you are working on a project for which there are other required projects, you can update both the main project and all dependent projects using the IDE's Update with Dependencies command.

To update an entire project and all dependent projects:

  1. Right-click the project and choose CVS > Update with Dependencies.

    2. The IDE incorporates any changes existing in the repository version of all of the projects with the local copies.
