The IDE includes tools that allow you to view and manage the evolution of changes in version-controlled files.

Viewing Versioning Information

The Versioning window displays the status and revision history of files in each VCS working directory registered with the IDE. Files are displayed in a tree view using icons to indicate their type and current status.

  • You can set whether or not the IDE displays file status annotations and at what level of detail in the Options window.

The Versioning window's file tree represents versioned files using the following icons:
Icon Description

General version-controlled file node, annotated with a revision number and log message.

Current revision node. The current copy of a file as it exists in your working directory.

Revision subnode. A prior revision of a file existing in the VCS repository.

Branch revision node. A prior revision of a file existing on a branch in the VCS repository.

Current VCS file status is indicated by adding the following badges to Versioning window file icons:
Badge Description

Up to Date badge. The local copy of the file is the same as the current repository revision.

Needs Checkout badge.

Locally Modified badge. The local copy of the file has been changed and needs to be committed.

Locally Removed badge. The local copy of the file has been removed from the working directory.

Merge Conflict badge. The repository copy of the file is a more recent revision than the local copy and contains changes which conflict with the local version's new edits.

To open the Versioning window, choose Window > Versioning > Versioning (Ctrl-8).

Monitoring VCS operations in the VCS Output Window

The VCS Ouput window automatically opens at the bottom of the IDE whenever you run VCS commands that are set to display output. Each operation opens in its own VCS Output tab which includes a command status bar and a Stop button that you can use to cancel active commands.

To open the VCS Output window, choose Window > Versioning > VCS Output (Ctrl-9).

Calling Version Control Commands

The VCS commands submenu contains all of the available VCS commands for the currently selected file or directory. The VCS commands submenu assumes the name of the working directory's VCS profile (CVS, for example) for the selected node.

You can access the VCS commands submenu by right-clicking a file or directory in any of the following:

  • Versioning window
  • Files window
  • Favorites window
  • VCS Groups window
  • Search Results window
  • Source Editor

    • The Status Bar in the bottom left of the IDE window displays the current command's status. If the command is set to display ouput, the output is automatically displayed in the VCS Output window.

  • By default, the IDE presents you with basic options when calling VCS commands. To access advanced command options, hold down the Control key while choosing from the VCS commands submenu. To enable advanced options for all VCS commands, set the Advanced Mode property to True in the working directory's property sheet.
