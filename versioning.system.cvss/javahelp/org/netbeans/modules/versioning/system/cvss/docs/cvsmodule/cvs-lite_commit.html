Committing Local Changes to a Remote Repository

See Also

Once your working copies of version-controlled files have been edited, you can then place your changes into the repository using the Commit command.

  • Project directories must be located within a local working directory that is registered with the IDE in order to call CVS commands on them.

To commit changes in local files to a remote repository:

  1. Select the files or directories you wish to commit in the Versioning window, Projects window, Files window, or Favorites window.
  2. Ensure that your local copies of the files are up-to-date by right-clicking and choosing CVS > Update before committing your changes.
  3. Right-click the files or directories you wish to commit and choose CVS > Commit. Note that if the files you want to commit do not already exist in the repository, the Commit command will add them.

    4. The CVS Commit dialog box appears prompting you to enter a message describing your changes.

Ignoring Files

The IDE enables you to determine which files within your working directory are committed to the remote repository when calling CVS commands. Using the Ignore command, you can tell the IDE to ignore those files and directories you do not want to place under version control.

To ignore local files in your working directory:

  1. Select the files or directories you wish to ignore in the Versioning window, Projects window, Files window, or Favorites window.
  2. Right-click the files or directories you wish to ignore and choose CVS > Ignore. Note that the Ignore command is unavailable for files that already exist in the repository.

    3. The IDE ignores the file or directory whenever CVS commands are called on them or the directories within which they are stored.
