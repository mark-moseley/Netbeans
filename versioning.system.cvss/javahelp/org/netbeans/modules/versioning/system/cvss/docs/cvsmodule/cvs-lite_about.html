About Version Control

Version control systems (VCS) aid developer groups working with shared source files in common repositories by managing file revision history information. The IDE's CVS support provides a means of effectively managing changes to version-controlled files.

In the IDE, you can call CVS commands on both files and directories in the Projects, Files, Versioning, and Favorites window. To work with CVS in the IDE, you need to register a local working directory. For more information, see Setting Up CVS.

The IDE's CVS Tools

The IDE provides several tools to simplify the process of working with version controlled files including:

  • Versioning Window. Provides an overview of the changes in version-controlled files.
  • Search History Window. Enables you to search for specific revisions of version-controlled files.
  • Graphical Diff Viewer. Enables you to compare differences in file revisions graphically.
  • Merge Conflict Resolver. Assists you in resolving conflicts that occur when merging file revisions.
  • Output Window. Provides automatic monitoring of active VCS operations.

Other CVS Features

The IDE provides several tools to simplify the process of working with version controlled files including:

  • Color Coding. Enables you to view the current status of version-controlled files.
  • Annotations. Enables you to view the revision and author information for each line of version-controlled files.
