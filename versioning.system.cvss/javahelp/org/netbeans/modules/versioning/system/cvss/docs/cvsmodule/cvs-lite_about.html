About Version Control

Version control systems (VCS) aid developer groups working with shared source files in common repositories by managing file revision history information. The IDE's CVS support provides a means of effectively managing changes to version-controlled files.

In the IDE, you can call CVS commands on both files and directories in the Projects, Files, Versioning, and Favorites window. To work with CVS in the IDE, you need to do is register a local working directory. For more information, see Setting Up CVS.

CVS-Lite Tools

The IDE also provides several tools to simplify the process of working with version controlled files including:

  • Versioning Window. Enables you to view the evolution of changes in version-controlled files.
  • Graphical Diff Viewer. Enbles you to compare differences in file revisions graphically
  • Merge Conflict Resolver. Assists you in resolving conflicts that occur when merging file revisions.
  • VCS Output Window. Provides automatic monitoring of active VCS operations.
