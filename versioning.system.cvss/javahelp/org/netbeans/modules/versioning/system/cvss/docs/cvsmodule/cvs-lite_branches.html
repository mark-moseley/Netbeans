Working with Branches

See Also

Using the IDE's CVS support you can work with branches that enable you to maintain different versions of an entire code base. A branch is created by attaching a branch tag to a file or folder in the repository. When you modify files in a branch it does not affect the files in the main code line (also known as the "trunk"). You can, however, merge any modifications in the branch back into the trunk later.

Checking out Branches

Once a branch is created, you can check out the branch to copy the files to your local working directory.

  • You must create a new local working directory within which to checkout the branch.

To check out a branch to a local working directory:

  1. Choose CVS > Checkout from the main menu.
  2. On the first page of the CVS Checkout wizard, specify the location of the CVS repository by choosing from the CVS Root drop-down menu. If you are unfamiliar with the syntax, click the Edit button and enter the required information for the local CVS working directory you wish to use.
  3. Enter your password in the Password field and click Next.
  4. If you are using a proxy, click the Proxy Configuration button and enter the required information. Click Next.
  5. On the Module to Checkout page, specify the files and directories you want to check out in the Module field or click the Browse button to choose from a list of all modules in the repository.
  6. Specify the branch, revision number, or tag for the checkout in the Branch field or click the Browse button to choose from a list of all branches in the repository.
  7. Specify the local working directory into which you want to check out the selected modules or branches. Alternately, you can click the Browse button to navigate to the desired directory on your system.
  8. Click Finish to check out the files.

    9. The IDE initiates the Checkout command for the branch. The IDE's Status bar indicates the progress of the files downloading from the branch to your local working directory.

Working in a Branch

When you are working in a branch, any changes you make are made in the branch and do effect the trunk. To merge files back to the trunk or another branch, you can use the Update command, specifying the revision tags in the advanced options.

To merge files from a branch:

  1. In the Projects, Files, or Versioning window, right-click the files or folders you want to merge to the trunk (or a different branch) and choose CVS > Merge Changes from Branch.
  2. In the CVS Update dialog box, select Merge from Branch.
  3. Specify the branch, revision number, or tag for the checkout in the Merge from Branch field or click the Browse button to choose from a list of all branches in the repository.
  4. Select any options you require and click Merge.

    When you click OK, the files from your branch directory are merged into your local working directory. Merge conflicts are indicated in the Versioning window by the Merge Conflict badge ( ).

  5. Resolve any merge conflicts in your local working directory resulting from the merge.
  6. Right-click the files or folders to commit and choose CVS > Commit.
  7. Click OK to commit your local files to the repository.

Creating a branch

Branches are created by adding a branch tag to a file revision or directory.

To create a branch:

  1. In the Projects, Files, or Versioning window, right-click the file or folder you want to be the root of your branch and choose CVS > Branch.
  2. In the Branch dialog, enter a Branch name, Tag name, and set any desired options.
  3. Click Branch.

    4. The IDE adds a branch tag to the selected file or directory and creates the branch.

  • When you add a branch tag, the branch is immediately created in the repository without requiring the Commit command.
