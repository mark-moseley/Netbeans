CVS Tasks: Quick Reference

See Also

The following table describes common CVS tasks.

To perform this task: Follow these steps:
Check out sources
  1. Choose Team > (CVS >) Checkout from the main menu.
  2. In the CVS Checkout wizard, specify the location of the CVS repository, enter your password and proxy (if required), and click Next.
  3. Specify the module and branch you want to check out in the Module and Branch fields, respectively.
  4. In the Local Folder field, specify the location on your computer where you want the checked out sources to reside.
  5. Click Finish to check out the files.
View versioning information
  1. Select a project, directory, or file that is in a CVS working directory
  2. Choose CVS > Show Changes from the right-click menu. The Versioning window opens to display versioning information on the selected item(s).
Update local file versions
  1. Select the project, file, or folder that you want to update
  2. Choose CVS > Update from the right-click menu.
Diff files
  1. Select the file that you want to perform a diff on.
  2. Choose CVS > Diff from the right-click menu.
Commit local changes
  1. Select the file or directory that you want to commit.
  2. Choose CVS > Commit from right-click menu.
Switch to a branch
  1. Select the file or folder you want to be the root of your branch and choose CVS > Switch to Branch from the right-click menu.
  2. In the Switch to Branch dialog, enter a Branch name or click Browse to see a list of available branches.
  3. Click Switch.
Create a branch
  1. Select the file or folder you want to be the root of your branch and choose CVS > Branch from the right-click menu.
  2. In the Branch dialog, enter a Branch name or click Browse to see a list of available branches.
  3. If you want to tag the file or directory, select the Tag Before Branching checkbox and enter a Tag name.
  4. If you want to switch to the new branch after creating it, select the Switch to This Branch Afterwards checkbox.
  5. Click Branch.
Apply a tag
  1. Select the file or folder and choosing CVS > Tag.
  2. Enter the tag name in the text field or click Browse to choose from a list of existing tags.
  3. If you want to ensure that the tag is not attached to local revisions that are out of date, select the Avoid Tagging Locally Modified Files checkbox.
  4. If you want to move an existing tag to the current revision in your local working directory, select the Move Existing Tags checkbox.
  5. If you want to remove the specified tag from the selected item, select the Delete Tag checkbox.
Merge revisions
  1. Select the files or folders that you want to merge into your local working copy and choose CVS > Merge Changes from Branch from the right-click menu.
  2. Specify whether to start the merge from the branch root, or from a specific revision or tag.
  3. Specify the end point of the merge (i.e. either the trunk a branch head, or until a specific branch or revision.
  4. Select the Tag trunk after Merge checkbox if you want to specify a tag for the merged item.
  5. Click Merge.
Create a patch
  1. Select the locally modified file or folder and choose Team > (CVS >) Export Diff Patch from the main menu.
  2. Enter a name for the patch file and specify the location where you want to save the patch.
  3. Click Export.
Apply a patch
  1. Select the locally modified file or folder and choose Team > (CVS >) Apply Diff Patch from the main menu.
  2. Type the path or navigate to the patch file you want to apply.
  3. Click Patch.
