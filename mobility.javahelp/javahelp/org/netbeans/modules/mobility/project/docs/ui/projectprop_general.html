Mobility Project Properties: General

See Also

Use the General Properties Page to:

  • view the basic properties of a Mobility project.
  • set the application version to advance incrementally with each build.
  • use or ignore preprocessing directives in your code.

Project Properties

The General project properties page displays the following properties:

  • Project Name. The name of the current project.
  • Project Folder. The Ant project name for the active project.
  • Project Sources Location. Location of the project source files.
  • Required Projects. A list of projects the active project on which the project depends.

These fields cannot be edited.

You can check or uncheck the Rebuild Required Projects check box. If this box is checked, the required projects will be rebuilt before the active project is built.

Application Versioning

The Application Version Number field displays the version number of the current build. This field cannot be edited.

The Application Version Counter field displays the next version number to be used. The default advance is 0.0.1. To advance the number beyond this, use the drop-down menu to select a new digit, or enter the value into the field. For example, changing the value to 3 results in a build number of 0.0.3. changing the value to 100 results in the version number 0.1.0.

Using the Preprocessor

This check box is checked by default. If you do not want to use preprocessor directives, uncheck this box.
