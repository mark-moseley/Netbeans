Source Code Synchronization (UML Modeling)

See Also

There are two types of code that can be in a UML Modeling source file: source code that is written by you or source code that is generated by the UML Modeling tool. Preference settings defined in domain templates determine which template files and output parameters are to be used for the code generation process. By default, the generated code is annotated with the @generated tag.

When source code for a UML model is regenerated, by default, the source code synchronization process will replace source code that have the @generated tag. If you manually remove the @generated tag, the source code synchronization process will no longer automatically replace the method that was previously generated.

In the Generate Code dialog, you can specify that merge markers be added to existing generated source code elements. You can also add the markers manually. Markers need to be in the following formats in order to be recognized:

  • //#[regen=yes/no] - Specifies whether the source code element can be regenerated. If this parameter is not present, the code element will not be regenerated
  • //regenBody=yes/no - Specifies whether the source code element's header can be regenerated or whether the header and the body can be regenerated
  • //id=<a unique id value> - Used to initially match elements (types, attributes, methods, or enumerated literals) in the newly generated file against the previous version of the source file.
