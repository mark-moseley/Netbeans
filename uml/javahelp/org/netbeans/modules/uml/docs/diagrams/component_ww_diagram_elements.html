Working With Component Diagrams

See Also

The Component diagram typically contains one or more of the following modeling elements:

  • Component
  • Interface
  • Port
  • Assembly Connector
  • Class
  • Artifact
  • Dependencies (can be any of the following types: Dependency, Realize, Usage, Permission, Abstraction, Implementation)

You can manipulate dependencies between the elements of Component diagrams in the following ways:

  • Working With Component Elements and Dependencies
  • Working With Ports and Interfaces
