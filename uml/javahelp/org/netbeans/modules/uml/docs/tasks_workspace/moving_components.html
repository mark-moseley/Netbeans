Moving Modeling Elements

You can move UML Modeling elements in the following ways:

  • Between package nodes in the Projects window
  • From the Projects window to a diagram that is open in the Diagram editor

In both cases, you can drag the element onto its destination. Thus, the Projects window can act as a library of easily accessible, reusable objects.

Moving Elements Within the Projects Window

You can drag modeling elements (such as Classes, Packages, and Actors) and some other elements (such as attributes and operations) from one project to another project in the Projects window. You can also nest elements by moving an element under another element in the project tree.

  • Not all elements can be nested; some restrictions apply.

To move elements within the Projects window:

  1. In the Projects window, select an element.
  2. Drag the element onto a project node or another element node.
    UML Modeling relocates the element to its new location in the tree.

Dragging Elements From the Projects Window Onto Diagrams

You can drag modeling elements from the Projects window to a diagram that is open in the Diagram editor.

To move onto a diagram an element that is listed in the Projects window:

  1. In the Projects window, select an element.
  2. Drag the element onto the receiving diagram window.
