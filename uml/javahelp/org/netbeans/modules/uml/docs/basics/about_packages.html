About Packages

See Also

Packages let you further specify how you want to organize your UML Modeling project. Within a project, packages can do the following:

  • Break up the project into logically separate subsystems. (This is code-dependent, and you might be using namespaces.)
  • Organize the project by diagram types.
  • Support any organization that suits your needs.

When you create a package, a green folder denoting the new package appears in the Projects window. You can drag elements and use cases onto any package from elsewhere in the current project to organize the objects in specific packages. Once dropped in a package, an element can be scoped by that package when placed in a diagram.
