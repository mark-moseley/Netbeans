Configuring Domain Templates Options (UML Modeling)

See Also

The Templates tab in the UML panel of the Options window enables you to customize domain template settings that are used during the UML Modeling's code generation process.

Domain templates, which you define, is the mapping of an element type and a stereotype to a particular code generation template file. The three domain templates currently available are Basic Class, Basic Interface, and Basic Enumeration, all of which are designed for plain old java objects (POJOs). The sole purpose of creating domain templates is to generate text from an existing UML model. The default output format is Java code, but it can be C++, JavaScript, XML, Ruby, etc. The output format is completely determined by the template files that are defined for a particular domain template.

Domain templates can be grouped into categories. You can create a category with as few or as many domain templates as you want. Categories of domain templates are displayed in tree form. Each domain template has a set of properties that can be customized.

To add a new domain template category:

  1. Select the Categories node and click Add.
  2. Select the node for the new category and press F2 or triple-click the node to change the default name of the new category.

To remove an existing domain template category:

  • Select the category to be deleted and click Remove.

To add a new domain template:

  1. Select the node for the category in which to add the domain template and click Add.
  2. Select the node for the new domain template and press F2 or triple-click the node to change the default name of the new category.

To remove a domain template:

  • Select the domain template to be deleted and click Remove.

To modify a domain template's properties:

Select the node for the domain template and modify any of the following properties:
  • Element Type. Specifies the domain template's model element type. You may select only one model element type from the drop-down list.
  • Stereotype. Specifies the custom defined stereotype for the domain template. Some examples are my-foo, ejb-stateless, or persistence.
  • Description. Provides a brief description of the domain template.
  • Output Parameters. Lists the template files and other output parameters to be used during the code generation process when an element is identified as being of this type of domain template. Click Add Template to define the following output parameters for a new template file:
    • Filename format. Customizes the filename with a prefix, suffix, or both with the use of the {name} token, where name is the model element name during code generation. The default (an empty value) will use the model element's name as the filename.
    • Extension. Customizes the extension to be used on the generated source file.
    • Folder Path. Specifies a custom sub-folder path of the model element package's sub-folder path. This field is usually left blank.
    • Template File. Filename of the template that will be used to generate the source code.
      The templates that populate this field's choices are accessible and editible by choosing Tools| > Templates from the main menu of the IDE and in the Template Manager dialog, expand the UML > Code Generation node. The folders under Code Generation folder do not dictate the Categories in the Domain Templates tree, but you might design it as such to be consistent.

    Select a row and click Edit to modify a set of existing output parameters for a template file. Select a row and click Remove to delete a set of output parameters.
