Creating User-Defined Patterns

When searching for an answer to your design problem, you might find that none of the standard patterns included in the UML Design Center provide the solution you are looking for. The UML Modeling feature gives you ways to create your own pattern, document it, apply it, and share it with others.

Before building your pattern, decide where to create it. Patterns can be created so that they appear right in your Projects window, or they can be created in the UML Design Center window. A pattern appearing in the Projects window can be applied only to elements in that project, unless the pattern is promoted to the UML Design Center. Patterns that are created or promoted to the UML Design Center can be applied to any project.

If you decide to create your pattern in the UML Design Center, see Creating User-Defined Projects. Otherwise, you can begin building your pattern so that it appears in the Projects window.

Creating a user-defined pattern involves three processes:

  1. Creating a diagram.
  2. Building the pattern.
  3. Documenting the pattern.
