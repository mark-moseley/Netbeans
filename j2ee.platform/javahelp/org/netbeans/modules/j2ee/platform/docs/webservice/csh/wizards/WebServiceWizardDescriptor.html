New Web Service Wizard

See Also 

The New Web Service wizard creates a new Web Service. You open the New Web Service wizard by right-clicking a web application, choosing New > File/Folder and selecting Web Service from the Web Services category.

In the Name and Location page, you have to set the following properties:

  • Web Service Name. Specifies the name of the web service. The name is used to name the three files that are created by the web service wizard. By default, they are named as follows:
    • NewWebService-config.xml
    • NewWebServiceImpl.java
    • NewWebServiceSEI.java
  • Location. Specifies the source root where the web service files will be created. By default, you only have one source root, the folder. To add additional source root, right-click the project node in the Projects window, choose Properties, click Sources, and define a new source package folder.
  • Package. Specifies the package name where the web service files will be housed.
  • Web Service Creation Type. Specifies the artifacts used to create the web service.
    • From scratch. Use this when you have no code to expose. Click Use existing code when you have a session bean in an EJB module or a class in a web application that you want to expose. Click Next and then select the component that you want to expose as a web service.
    • From Local WSDL File. Browse to a WSDL file on your filesystem.
    • From WSDL URL. Specify a URL to an online WSDL file.
