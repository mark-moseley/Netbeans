New Web Service Client Wizard

See Also 

The New Web Service Client wizard creates a new Web Service Client. You open the New Web Service Client wizard by right-clicking a web application or Java application, choosing New > File/Folder and selecting Web Service Client from the Web Services category.

In the Specify Web Service Client Information page, you have to set the following properties:

  • Source of the WSDL description:
    • Running Web Service. Specifies the URL to a running web service's WSDL file. If you use a proxy server, you must click Proxy Settings and set the proxy host and port number, otherwise the IDE will not be able to download the WSDL file. Click Retrieve WSDL and, if the WSDL file's name automatically appears in the Local Filename field, the WSDL file is successfully downloaded.
    • Existing WSDL File. Specifies a WSDL file in your local filesystem. Click Browse to locate it in your filesystem.
  • Package. Specifies the package where the client stubs and other artifacts will be generated. Note that you will not be able to see this package and the client files in the Projects window, because they are re-generated whenever you build the project and deleted when you clean the project. Files of this nature are never displayed in the Projects window by default. To override this behavior and display the client files in the Projects window anyway, right-click the project node, choose Properties, and use the Sources panel to add the package containing the client files. The package will then be displayed in the Projects window, but beware of creating files in this package, because the IDE will re-generate it when the project is built and delete it when the project is cleaned.
  • Client Type. Specifies the type of client stubs that will be created by the wizard.
    • J2EE Container-generated static stub. This type is based on JSR-109, which enhances JSR-101 by defining the packaging of web services into standard J2EE modules, including a new deployment descriptor, and defining web services that are implemented as session beans or servlets. This is the recommended and portable (via J2EE 1.4 specification) type. When you choose this type, the IDE adds deployment information in the deployment descriptors and the container does the generation of the stub and other artifacts.
    • IDE-generated static stub. This type is based on JSR-101, which defines the mapping of WSDL to Java and vice versa. It also defines a client API to invoke a remote web service and a runtime environment on the server to host a web service. This type is not portable. When you choose this type, the IDE generates the stub and other artifacts, packages them in the archive, and deploys them.
