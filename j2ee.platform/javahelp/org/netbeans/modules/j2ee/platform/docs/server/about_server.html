About Servers

See Also

A server is a computer that serves up applications. Any computer can be turned into a server if you install server software and connect the computer to the Internet. Server software for the following server types are supported by the IDE:

  • SJS App Server

    Use this server for the following development activities:

    • Creating and deploying enterprise applications and enterprise beans.
    • Creating and deploying JSR-109 and JSR-101 web services.
    • Libraries included with the SJS App Server can be used to compile web services and clients, verify that your application implements the J2EE 1.4 specification correctly, create a new web application with JavaServer Faces (JSF) support, or add JSF support to an existing web application
  • Tomcat Web Server

    Use this server for the following development activities:

    • Creating and deploying web applications.
    • Deploying JSR-101 web services.

Servers in the IDE

Before you can deploy an enterprise application, web application, JSP, servlet, or EJB module, the server to which you are going to deploy needs to be registered with the IDE. By default, only the bundled Tomcat Web Server is registered with the IDE. You must register all other servers yourself. When a server is registered with the IDE, you can see its node in the Runtime window under the the Server Registry node. To target a server for a project, right-click it, choose Properties, click Run, and select the server.
