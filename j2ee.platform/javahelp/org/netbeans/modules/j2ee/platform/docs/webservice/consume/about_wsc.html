About Web Service Clients

A web service client is created to consume (i.e., use) a specific web service. The way in which a web service client consumes a web service depends on the way in which the provider makes the web service available:

  • The provider distributes a URL to the WSDL file of a running web service.
  • The provider distributes a WSDL file, which is available to you on your local filesystem.

There are two types of web service clients, JSR-109 and JAX-RPC:

  • J2EE 1.4 (JSR-109). Enhances JSR-101 by defining the packaging of web services into standard J2EE modules, including a new deployment descriptor, and defining web services that are implemented as session beans or servlets. This is the recommended and portable (via J2EE 1.4 specification) type.
  • JAX-RPC Static Stub. Defines the mapping of WSDL to Java and vice versa. It also defines a client API to invoke a remote web service and a runtime environment on the server to host a web service.

Web Service Clients in the IDE

In the IDE, when you create a web service client, deployment information is generated in the deployment descriptor (the web.xml file) and a compilation target is generated in the Ant build script (the build.xml file). After you finish developing the web service client, you can use the Web Service Clients panel in the Project Properties dialog box to specify how you want to build it. Then, when you build the web service client, the wscompile tool is called to compile it.
