Deploying and Undeploying an EJB Module

See Also

Unless your EJB module is designed to be deployed as a stand-alone EJB module, you should always deploy EJB modules by deploying the enterprise application that contains them. If you run the Run Project or Deploy commands on an individual EJB module that is part of an enterprise application, the IDE only deploys that EJB module. Any changes you have made in the other modules in the project are not propagated to the server.

To deploy or undeploy a single EJB module:

  • In the Projects window, right-click the EJB project and choose one of the following:
    • Run Project. Starts the target application server and domain and deploys the EJB module.
    • Deployment  Deploy. Deploys the EJB module to the target application server and domain.
    • Deployment  Undeploy. Removes the EJB module to the target application server and domain.
    • Deployment  Redeploy. Undeployes the EJB module and then deploys it again.
