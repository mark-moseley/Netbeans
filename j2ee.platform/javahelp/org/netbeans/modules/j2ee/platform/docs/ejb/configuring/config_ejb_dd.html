Configuring EJB Deployment Descriptors

See Also

Deployment descriptors are XML-based text files whose elements describe how to assemble and deploy a module to a specific environment. The elements also contain behavioral information about components not included directly in code.

If you are developing enterprise beans on the Java EE 5 platform and deploying to a Java EE 5 compliant application server, you develop enterprise beans using EJB 3.0 technology. EJB 3.0 technology enables you to use annotations in your classes instead of deployment descriptors to describe behavior.

For EJB modules, there are two types of deployment descriptors:

  • The general J2EE deployment descriptor that configures deployment settings on any J2EE-compliant implementation. The general EJB deployment descriptor is called ejb-jar.xml. The IDE provides a graphical editor for the general deployment descriptor and automatically updates it when you edit your code.
  • The server-specific deployment descriptor that configures deployment settings for the application server to which you are deploying. For the Sun Java System Application Server, the IDE automatically updates the deployment descriptor as you edit your files. The IDE also provides a graphical deployment descriptor editor to edit some descriptor files. For all other application servers, you have to write the server-specific deployment descriptors yourself.

To edit ejb-jar.xml:

  1. In the Projects window, expand the Configuration Files node for your EJB module project.
  2. Double-click ejb-jar.xml to open the file in the graphical editor.
  3. Select one of the following tabs:
    • General. This tab enables you to specify general information about the EJB module, details about security roles and descriptor details about each of the enterprise beans in the module. See the following topics for more information about the properties you can set in the General tab:
      • ejb-jar.xml Visual Editor - Module Details
      • ejb-jar.xml Visual Editor - Security Roles
      • ejb-jar.xml Visual Editor - Session Beans
      • ejb-jar.xml Visual Editor - Entity Beans
      • ejb-jar.xml Visual Editor - Message-Driven Beans
    • CMP Relationships. This tab enables you to view and modify CMP relationships for CMP beans in your module. See the following topic for more information about the properties you can set in the CMP Relationships tab:
      • ejb-jar.xml Visual Editor - CMP Relationships
    • XML. This tab enables you to edit the descriptor file in the XML editor.
  4. Edit the properties.
  5. Save your changes.

To edit sun-ejb-jar.xml:

  1. In the Projects window, expand the Configuration Files node for your EJB module project.
  2. Double-click sun-ejb-jar.xml to open the file in the graphical editor.
  3. Double-click sun-ejb-jar.xml to open it in the graphical editor.
  4. Select one of the following tabs:
    • General.
    • EJB.
    • Security.
    • Web Services.
    • Messaging.
    • XML.
  5. Edit the deployment descriptor as necessary. See the following topics for more information:
    • sun-ejb-jar.xml Visual Editor: Sun Configuration
    • sun-ejb-jar.xml Visual Editor: Session Bean
    • sun-ejb-jar.xml Visual Editor: Entity Bean
    • sun-ejb-jar.xml Visual Editor: Message Bean
  6. Choose File > Save to save your changes.
  7. Right-click sun-ejb-jar.xml and choose Edit to edit the deployment descriptor's XML source. You can only open in this mode if the file is not already open in the visual editor. The Source Editor provides code completion and validation for all XML files.

To edit the server-specific deployment descriptor for any other application server:

  1. Double-click the deployment descriptor node.
    The IDE opens the XML source of the deployment descriptor in the Source Editor.
  • If you enter any XML syntax errors the IDE automatically alerts you. To fully ensure your changes have not cause any errors, you should verify the EJB module.
