ejb-jar.xml Visual Editor: Bean Environment

See Also 

The ejb-jar.xml visual editor lets you edit the general for an EJB module. You open the editor by expanding the project's Configuration Files node in the Projects window and double-clicking ejb-jar.xml. The Enterprise Beans section of the General view contains a section for each of your EJB module's enterprise beans.

The section for each enterprise bean has a Bean Environment section. The Bean Environment section lets you specify the following information:

Enterprise Bean References

Lists all of the enterprise beans that your bean references. When you add a bean reference, the IDE creates lookup code for in the bean class of the enterprise that is doing the calling, and adds a reference to the bean in the deployment descriptor.

The easiest way to create a bean reference is to open a bean class in the Source Editor, right-click in the bean class, choose Enterprise Resources  Call Enterprise Bean.

Environment Entries

In the Environment Entries section, you can define properties in the deployment descriptor that you can then access in the code of your EJB module. For example, an entry name of a enterprise bean that calculates money exchanges might be PoundToDollar and the value for this entry might be 1.55, which indicates the pound-to-dollar exchange rate.

The environment entry values may be one of the following Java types: String, Character, Integer, Boolean, Double, Byte, Short, Long, and Float. Make sure that each entry's value matches its type. Also make sure that the name for each entry is unique in the entire deployment descriptor.

Resource References

In the Resource References section, you can define references that your bean needs to access resources such as the following:

  • JMS data sources
  • JDBC data sources
  • Connectors to enterprise information systems
  • Connectors to message ques
  • JavaMail resources

JMS data sources and JDBC data sources are registered automatically when you use the Send JMS Message and Use Database commands.

For each resource reference, you need to do the following:

  • Type the name of the resource, prepended with either jdbc/, mail/, jms/, env/ or url/, in the Name column.
  • Type Application or Container in the Authentication column to indicate whether you want the authentication of resource users to be managed by the enterprise bean or the container.
  • Type one of the following resource types in the Resource Type column:
    • javax.sql.DataSource. Represents a data store like a database.
    • javax.mail.Session. One of the following:
      • A mail store, where mail messages are stored in folders
      • A mail transport used to send a mail message
    • java.net.URL. Represents a file resource.
    • javax.jms.QueueConnectionFactory. Rrepresents a connection factory for connecting to a Queue destination.
    • javax.jms.TopicConnectionFactory. Represents a connection factory for connecting to a Topic destination.
    • javax.jms.ConnectionFactory. Rrepresents a connection factory for connecting to a JMS provider.
    • javax.resource.cci.ConnectionFactory. Represents a connection factory for connecting to an EIS.
    • javax.xml.registry.ConnectionFactory. Represents a connection factory for connecting to a JAXR registry.
  • (Optional) Type Shareable or Unshareable in the Shareable column to specify whether connections obtained through the given resource manager connection factory reference can be shared.
  • (Optional) Type a description of the resource refenerence in the description column.

Resource Environment References

Resource environment references are another type of environment entries. They allow you to refer to administered objects that are associated with resources (for example, JMS Destinations), by using logical names. You can type the name, resource type, and optionally a description for each resource environment reference.

Security Roles References

In the Security Roles References section, you can link a reference name used in your code to a security role defined on the application server. The value in the Name column must be the security role name that is used as a parameter to the isCallerInRole(String roleName) method in your enterprise bean. You can also optionally link the role name to a security role that is defined ... (where? in the bean or on the server?)

All enterprise beans also have the following sections in the ejb-jar.xml visual editor:

  • Enterprise Bean Implementation and Interfaces
  • Bean Details
