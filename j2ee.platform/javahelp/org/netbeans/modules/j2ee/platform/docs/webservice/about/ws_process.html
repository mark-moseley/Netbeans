Working with Web Services

The following table shows how to work with web services.

Design the web service
  1. Go to Help > Java BluePrints Solutions Catalog.
  2. Read topics under the "Web Service Design" item.

Create the web service
  1. Choose the implementation form for your web service.
  2. Do one of the following:
    • Create a web service from scratch. When creating a web service from scratch, you can use existing code to implement your service.
    • Create a web service from a WSDL File

Develop the web service
  1. In the Projects window or Files window, double-click the files that you would like to edit.
  2. Use the Source Editor to develop the web service. When you right-click in the Source Editor, you can use the IDE to generate skeleton code for one or more of the following:
    • Add a web service operation
    • Add a SOAP message handler
    • Call an EJB
      Since the WSDL file governs the interface to the web service, you may not add new operations to web services that are created from a WSDL file, because these operations will not be reflected back in the WSDL file.

Build the web service
  1. Do one of the following:
    • If the module within which the web service is implemented is part of a J2EE application, make sure the J2EE application is set as the main project and choose Build > Build Main Project (F11).
    • If the project is a stand-alone module, or if you want to build the module without building the other modules in the J2EE application, right-click the project's node in the Projects window and choose Build Project.
  2. Analyze the result and, if necessary, customize the wscompile tool's features, depending on your needs:
    • Building a Web Service from a WSDL File
    • Building a Document-Style Web Service
    • Building a RPC-Style Web Service

Deploy the web service

You can deploy a web service as a stand-alone module or as a part of a J2EE application.

  1. To deploy it as part of a J2EE application, right-click the J2EE application's J2EE Modules node, choose Add J2EE Module, select the module that contains the web service, and click OK.
  2. Make sure the appropriate server is targeted.
  3. Choose Run > Run Main Project or right-click the project and choose Run Project.

Test the web service
  1. Register the web service in the IDE .
  2. Test the web service from the IDE.

Consume the web service
  1. Create the web service client.
  2. Test the web service from the IDE.
  3. Once you are sure that the web service is useful to you, use the Source Editor to develop the web service client. When you right-click in the Source Editor, you can use the IDE to generate skeleton code for calling a web service operation.
  4. Build the web service client.
  5. Make sure the appropriate server is targeted.
  6. Choose Run > Run Main Project or right-click the project and choose Run Project.
