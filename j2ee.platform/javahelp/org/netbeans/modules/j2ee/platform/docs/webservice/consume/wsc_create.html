Creating a Web Service Client

See Also 

  1. Create a web application project.
      Later, you will not be able to build the web service client unless the SJS Application Server is the target server for the project. Therefore, take the appropriate steps to prepare to develop web services and clients.
    If the WSDL file imports schemas, other WSDL files, or both, from the local file system, the web service client will only work if the imported schemas and WSDL files are copied beforehand into the WEB-INF/wsdl (or META-INF/wsdl) folder. If you do not do this, the Web Service Compiler will not be able to locate these imported files at the end of the procedure below.
  2. From the Projects window or Files window, right-click a node in the web application project and choose New  Web Service Client.
  3. Depending on what the provider has distributed, do the following:
    • To generate a client from a running web service, type the web service's URL and specify a package where the client is to be generated.
    • To generate a client from a WSDL file on your local filesystem, browse to the WSDL file and specify a package where the client is to be generated.
  4. Specify a package name for the generated interfaces that will be used by your user code to access and interact with the web service.
  5. Specify the web service client type:
    • J2EE 1.4 (JSR-109). Enhances JSR-101 by defining the packaging of web services into standard J2EE modules, including a new deployment descriptor, and defining web services that are implemented as session beans or servlets. This is the recommended and portable (via J2EE 1.4 specification) type.
    • JAX-RPC Static Stub. Defines the mapping of WSDL to Java and vice versa. It also defines a client API to invoke a remote web service and a runtime environment on the server to host a web service.
  6. Click Finish.

    7. The IDE creates a node for the web service under the Web Service References node in the Projects window. For each web service operation defined in the WSDL file, the IDE creates a sub-node with the operation's name. Right-click an operation node and choose Test Operation to test the operation directly within the IDE, without writing a single line of code. In addition, the IDE does the following:

    • Creates a folder, if not already present, named wsdl under web/WEB-INF to contain the WSDL file.
    • Adds a service-ref element to the deployment descriptor (the web.xml file) for this service.
    • Adds a wscompile target for this service to the Ant build script (the build.xml file).
    • Shows the service in the Web Service registry in the Runtime tab.
