Working with Enterprise Applications

The following table shows how to work with enterprise applications.

Create the application and module projects
  1. Create the project for the enterprise application.
  2. Create the projects for any EJB modules.
  3. Create the projects for any Web modules.
  4. Add the modules to the enterprise application.

Code the modules
  1. Create servlets and JSP files in the web modules.
  2. Create new session beans and message-driven beans.
  3. Generate entity beans from existing databases.
  4. Call enterprise beans from web modules and other enterprise beans.
  5. Use server resources like databases and JMS messages.

Configure the application
  1. Edit the enterprise application's deployment descriptors.
  2. Select libraries to bundle with the EAR file.
  3. Set runtime options like the application's default Web page and target application server.

Build the project
  1. Choose Build   Build Main Project or right-click the project and choose Build Project.

Deploy the application
  1. Right-click the enterprise application project and choose Run Project to start the application server and deploy the entire application.
  2. Right-click the project and choose Redeploy Project to undeploy the enterprise application and deploy it again.
  3. Right-click any of the application's module projects and choose Deploy Project to deploy the module without deploying the other modules in the application.
