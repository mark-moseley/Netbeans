Creating a Web Service from Scratch

See Also 

  1. Do one of the following:
    • Create a web application project.
    • Create an EJB module project.
  2. In the Projects window or Files window, right-click the project node and choose New  Web Service.

    3. The New Web Service wizard opens.

  3. Type the web service name and specify a package to house the web service.
  4. Click From scratch.
  5. Click Finish.

    The IDE creates the following:

    • The Service Endpoint Interface (the "interface"), which declares the web service's operations. In the IDE, the interface is identified by the suffix "SEI". For example, if you named your web service CalculatorWS, its interface is named CalculatorWSSEI. The IDE automatically populates the interface with a declaration for each web service operation added by means of the Add Operation dialog box.
    • The class for the implementation of the interface. The IDE creates a skeleton operation to the implementation class when you use the Add Operation dialog box. For example, if you are creating a web service that adds two integers, the skeleton operation that the IDE creates in the implementation class is as follows: 
          public int add(int a, int b) throws java.rmi.RemoteException {
       
    }

      In the IDE, an implementation class is identified by the suffix "Impl" (for web applications) or "Bean" (for EJB modules). For example, if your web service is named CalculatorWS, its implementation class is named CalculatorWSImpl or CalculatorWSBean.

    By default, both files are housed in a package within your project's src folder (in the Files window) and you can access it in the Source Packages node (in the Projects window).
  6. Use the Source Editor to develop the web service. When you right-click in the Source Editor, you can use the IDE to generate skeleton code for one or more of the following:
    • Add a web service operation
    • Add a SOAP message handler
    • Call an EJB
    • Use a database
    • Send a JMS message
      For help on designing and developing your web services, go to Help  BluePrints Solutions Catalog. Select topics under the "Service Oriented Architectures with Web Services" item.
  7. Generate the WSDL file by building the web service as a document-style web service or by building the web service as a RPC-style web service .
  8. Optionally, tweak the WSDL file by hand.
