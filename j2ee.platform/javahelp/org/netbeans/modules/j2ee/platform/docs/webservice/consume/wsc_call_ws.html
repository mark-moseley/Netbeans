Calling a Web Service Operation

See Also 

A web service operation can be called from a web application, an EJB module, or a MIDP client (MIDlet).

To call a web service operation:
  1. Create a web service.
  2. Add a web service operation.
  3. Create a web service client or create an EJB module.
  4. If the web service client is deployed from a web application, create a servlet.
  5. Right-click anywhere in a class in the Source Editor and choose Web Service Client Resources  Call Web Service Operation.
      This command is only enabled when you click within the body of the class (after the class declaration).
  6. Expand the nodes and select a web service operation.
      The OK button is only enabled when you select a valid web service operation.
  7. Click OK.

    The IDE adds the following to the servlet:

    • A try-catch block containing the method invocation.
    • A getXXX method (where XXX is the name of the web service).
    • A getXXXSEIPort method (where XXX is the name of the web service).
    • Two instance variables to represent the service and the port.
    • The necessary import statements.
      Note that the Source Editor underlines the generated code as containing errors. This is because the IDE has not generated the classes imported by these methods. After the first time you build the project the error underlining disappears.
