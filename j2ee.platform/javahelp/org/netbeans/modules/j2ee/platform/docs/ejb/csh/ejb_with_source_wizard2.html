New Enterprise JavaBeans Module from Existing Sources Wizard: Existing Sources

See Also 

The New Enterprise JavaBeans Module from Existing Sources wizard creates a new   containing an EJB module. You open the New Empty Enterprise Application wizard by opening the New Project wizard and selecting the Enterprise JavaBeans Module from Existing Sources template from the Enterprise category.

Your existing EJB module source does not have to adhere to any particular directory structure. You specify the locations of the configuration files, libraries, and source roots. The only requirement is that the module contain a valid ejb-jar.xml deployment descriptor.

In the Existing Sources page, you specify each of the project's source package folders (containing source packages) and test package folders (containing JUnit tests). You cannot have two projects with the same source package folder open at the same time. If you want to add a source package folder that is already in an open project to a new project, you have to create the project without the source folder, close the existing project, and then add the source package to the folder in the Project Properties dialog box.
