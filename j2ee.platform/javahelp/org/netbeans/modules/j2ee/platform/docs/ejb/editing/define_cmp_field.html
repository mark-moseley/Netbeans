Defining a CMP Field

See Also

An entity bean represents data in a database. If it uses container-managed persistence (CMP), the bean must have a set of fields that the container uses to map to fields in the database. One of these fields, or a combination of them, can be a  that uniquely identifies records in the database.

If you use the Entity Bean wizard to create the bean, you can have the wizard create the CMP fields for you from an existing database or database schema. You can also create new CMP fields and edit the existing CMP fields for any existing entity bean.

When you add a new CMP field to an entity bean, the IDE does all of the following:

  • Adds the ... method to the bean class and the ... to the ...
  • Registers the CMP field in the deployment descriptor.

To define a CMP field for an existing entity bean:

  1. In the Projects window, expand the Configuration Files node for your EJB module project.
  2. Double-click ejb-jar.xml. The graphical editor opens in the Details view.
  3. Expand the entity bean header and the CMP fields header. The table lists all of the existing CMP fields.
  4. Click Add or Edit to make changes to the list of CMP fields.
