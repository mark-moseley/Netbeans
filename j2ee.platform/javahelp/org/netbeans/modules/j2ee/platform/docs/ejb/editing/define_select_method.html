Defining a Select Method

See Also

Select methods are a type of query method similar to finder methods, and, like finder methods, they use EJB QL. A select method is more flexible than a finder method and is visible only to the bean class; therefore, a select method can be used only by methods of the bean class. A select method inherits the transaction context of the method that calls it. In addition to returning a bean or a collection of beans, a select method can also return the value of a field.

To define a select method for an entity bean:

  1. Open the EJB module's editor by...
  2. Expand the entity bean's node and the CMP Finders and Selects node. The table lists all of the existing CMP finder methods and select methods.
  3. Click Add or Edit to make changes to the list finder methods and select methods.
  4. In the Add New Select Method dialog box, define the method signature:
    1. Add text to the base ejbSelect name of the method.
    2. Choose the method's return type from the combo box:
      • If the method returns the value of a CMP field, choose the appropriate Java or primitive data type, such as java.lang.String or int.
      • If the method returns an enterprise bean, choose the bean's type from the combo box.
      • If the method returns more than one bean, choose one of the Set types if returned beans cannot be duplicates, or choose one of the Collection types if returned beans can be duplicates.
    3. Specify any parameters that the method needs.
    4. Add any additional exceptions to throw.
    5. Enter the EJB QL statement that the method will use to find the data that it is to return.
  5. Click OK to create the method.
  6. The IDE adds the method signature to the bean class.
