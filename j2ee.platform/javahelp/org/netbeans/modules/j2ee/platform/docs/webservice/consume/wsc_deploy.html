Deploying a Web Service Client

See Also 

Unless the module (EJB module or web application module) in which the web service client is implemented is designed to be deployed as a stand-alone module, you should always deploy it by deploying the enterprise application that contains it. If you run the Run Project or Deploy Project commands on an individual module that is part of an enterprise application, the IDE only deploys that module itself. Any changes you have made in the other modules in the project are not propagated to the server.

To deploy a client from within an enterprise application project:

  1. In the Projects window, add the web application module or add the EJB module to the enterprise application.
  2. Right-click the enterprise application project and choose one of the following:
    • Run Project. Only takes the minimal steps needed to run the project. For example, if you only changed one file, only that file is sent to the server.
    • Deploy Project. Undeploys and then deploys the complete module no matter what changes you made, even if none.

To deploy a client from within a module project:

  • Right-click the module project and choose one of the following:
    • Run Project. Only takes the minimal steps needed to run the project. For example, if you only changed one file, only that file is sent to the server.
    • Deploy Project. Undeploys and then deploys the complete module no matter what changes you made, even if none.
