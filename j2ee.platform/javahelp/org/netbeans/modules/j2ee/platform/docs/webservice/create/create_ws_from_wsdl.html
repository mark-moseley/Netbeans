Implementing an Existing WSDL File

See Also 

When you and your business partners agree on a "contract" in the form of a WSDL file, you can use the IDE to implement it. You can also use the IDE to reimplement web services that have been published elsewhere, such as those offered by Amazon.com. You do not need to know the details of the WSDL file when you create the web service, because the IDE provides wizards that guide you through the process.

To implement a WSDL file:

  1. Do one of the following:
    • Create a web application project.
    • Create an EJB module project.
      Later, you will not be able to build the web service unless the SJS Application Server is the target server for the project.
  2. In the Projects window or Files window, right-click the project node and choose New  Web Service.

    3. The New Web Service wizard opens.

  3. Type the web service name and specify a package to house the web service.
  4. Do one of the following:
    • Browse to a WSDL file on your filesystem.
    • Specify a URL to an online WSDL file.
  5. Click Finish.

    The IDE does the following:

    • It "normalizes" the WSDL file. For example, a binding element is created if it did not exist in the WSDL file. Also, the service element is created to be compatible with the name that you specified in the wizard.
    • It creates the Service Endpoint Interface (the "interface"), which declares the web service's operations. In the IDE, the interface is identified by the suffix "SEI". For example, if you named your web service CalculatorWS, its interface is named CalculatorWSSEI. The IDE automatically populates the interface with a declaration for each web service operation from the WSDL file.
    • It creates the class for the implementation of the interface. The operations found from the WSDL file are added to this class. In the IDE, an implementation class is identified by the suffix "Impl" (for web applications) or "Bean" (for EJB modules). For example, if your web service is named CalculatorWS, its implementation class is named CalculatorWSImpl or CalculatorWSBean.
    By default, both files are housed in a package within your project's src folder (in the Files window) and you can access it in the Source Packages node (in the Projects window).
  6. Build the web service.
