Building an EJB Module

See Also

There are two ways to build an EJB module:

  • As a stand-alone EJB module. The IDE packages the necessary JAR files with the EJB module JAR file.
  • As part of a J2EE application. The IDE packages all of the necessary JAR files as part of the J2EE application and not as part of the individual modules.

To build a stand-alone EJB module:

  • Right-click the EJB module project in the Projects window and choose Build Project. Or, if the EJB module project is the main project, choose Build  Build Main Project (F11).

When you build a EJB module project as a stand-alone module, the IDE does the following:

  • Compiles the classes to the project's build/jar folder.
  • Places any JAR files that are one your project's classpath and are scheduled for inclusion in deployment into the build/jar folder.
  • Builds the EJB module's JAR file to the EJB module's dist folder. The JAR file includes any classpath items scheduled for inclusion in deployment.

To build an EJB module as part of an enterprise application:

  1. In the Projects window, expand the enterprise application's J2EE Modules node and make sure that the EJB module is listed. If you have not added the EJB module to the enterprise application, right-click the J2EE Modules node, choose Add J2EE Module, and select the EJB module.
  2. Right-click the enterprise application project and choose Build Project. Or, if the enterprise application project is the main project, choose Build  Build Main Project (F11).

When you build an EJB module as part of an enterprise application, the IDE does the following:

  • Compiles the classes to the project's build/jar folder.
  • Places any JAR files that are one your project's classpath and are scheduled for inclusion in deployment into the enterprise project's build folder.
  • Builds the EJB module's JAR file to the EJB module project's dist folder and to the enterprise application project's build folder. The EJB modue JAR file does not contain any of the classpath elements.
  • Builds the enterprise application EAR file to the enterprise application project's dist folder.

Notes:

  • The information in this page only applies to . are built according to the instructions in your project's Ant script.
  • You can mark a classpath item for inclusion in deployment in the EJB module's Project Properties dialog box. Right-click the module's Libraries node and choose Properties, then select the checkbox for each classpath item you want to include in deployment.
