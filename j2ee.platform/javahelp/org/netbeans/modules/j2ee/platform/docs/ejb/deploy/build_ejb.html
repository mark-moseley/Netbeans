Building an EJB Module

See Also

In general, you build an EJB module project just as you would build any other project. The main difference is whether you want to package the JAR files on the project's classpath with the EJB module JAR file.

If you are deploying the EJB module as a stand-alone module, you generally want to package the necessary JAR files with the EJB module JAR file. If, on the other hand, the module is part of a J2EE application, you should package all of the necessary JAR files as part of the J2EE application and not as part of the individual modules.

The IDE automatically detects whether an EJB project is part of a J2EE application and sets whether to package the JAR files on the classpath accordingly. The IDE does not package API libraries like the Servlet API in your EJB module JAR file. You can adjust which JAR files are packaged with your EJB module in the EJB module project's Project Properties dialog box.

How do you remove JAR files from a J2EE application if you're planning on making them available to all applications and modules deployed on the server?

To build an EJB module, do one of the following:

  • If the EJB module is part of a J2EE application, make sure the J2EE application is set as the main project and choose Build  Build Main Project (F11).
  • If the project is a stand-alone EJB module, or if you want to build the EJB module without building the other modules in the J2EE application, right-click the project's node in the Projects window and choose Build Project.

To define which JAR files are packaged with the EJB module JAR file:

  1. In the Projects window, right-click the project's node and choose Properties.
  2. Click Compiling Sources in the left panel of the Project Properties dialog box.
  3. In the Compilation Classpath list, select the checkbox for each classpath item that you want to package with the EJB module JAR file.
