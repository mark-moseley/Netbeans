Defining a Finder Method

See Also

A finder method is a type of query method. (Select methods are the other type of query method.) Finder methods are unique to entity beans because entity beans represent data in a database that can be identified and found. Session beans do not represent database data and therefore do not have data that can be found.

A typical finder method for an entity bean is the required findByPrimaryKey method that the entity bean wizard creates for you when you define a primary key for your entity bean.

To define a finder method for an entity bean:

  1. In the Projects window, expand the Enterprise JavaBeans node.
  2. Right-click enterprise bean's node and choose Add  Add Finder Method.

    You can also open the bean class and right-click anywhere inside the body of the class and choose Enterprise JavaBeans (EJB)  Add Finder Method. This command is only enabled when you click within the body of the class (after the class declaration).

  3. In the New Finder Method dialog box, define the method signature:
    1. Add text to the base find name of the method.
    2. Specify the return cardinality for the method. This setting should match the return cardinality of the statement in the database (one-to-one, one-to-may). If you specify One, the statement returns the component interface. If you set Many, the statement returns a collection object.
    3. Enter the EJB QL statement that the method will use to find the enterprise bean or beans that the method is to return.
    4. Specify which interfaces to add the method to (remote and/or local).
    5. Specify any parameters that the method needs.
  4. Click OK to create the method.
  5. The IDE adds the method signature to the home interface or local home interface or both, depending on the bean's component interface set.
    • If the entity bean uses container-managed persistence (CMP), you are done. The container implements the finder method from the signature in the home interface or local home interface, so the home interface and the local home interface are the only places the method appears.
    • If the entity bean uses bean-managed persistence (BMP), the IDE adds an ejbFindyourname method to the bean class so you can implement the method. You must implement the ejbFindyourname method yourself.
When the IDE adds a method to the bean class, the IDE does not save the class. Only when you compile or save the class is the new method saved to disk.

To view or edit existing finder methods:

  1. In the Projects window, expand the Configuration Files node for your EJB module project.
  2. Double-click ejb-jar.xml. The graphical editor opens in the Details view.
  3. Expand the entity bean's header and the CMP Finders header. The table lists all of the existing CMP finder methods.
  4. Click Add or Edit to make changes to the list finder methods.
