Creating an EJB Module Project

See Also

You develop enterprise beans in the IDE by creating an EJB module project. An EJB module project creates one EJB module JAR file, which can contain multiple enterprise beans. The IDE maintains information about the EJB module in the module's deployment descriptors.

You can create an EJB module by doing any of the following:

  • Creating a new Enterprise Application project with an empty EJB module project.
  • Create an empty EJB module project. You can manually add the EJB module to an enterprise application or deploy it as a stand-alone EJB module.
  • Import existing enterprise bean code into a standard project. A standard project uses an IDE-generated build script to build, run, and debug your project.

    You can import an individual EJB module using the Enterprise JavaBeans Module with Existing Sources template, or import an entire enterprise application using the Existing Blueprint Application template. For Existing Blueprint Application projects, the project's folder structure must adhere to the J2EE Blueprints recomendations. For Enterprise JavaBeans Module with Existing Sources projects, the module can have any folder structure.

  • The IDE only supports J2EE 1.4 EJB modules and automatically updates all J2EE 1.3 EJB modules to J2EE 1.4. The J2EE 1.4 specification is fully backwards-compatible, so upgrading the specification level does not change the behaviour of your code.

To create an EJB module project:

  1. Choose File  New Project (Ctrl-Shift-N) from the main window.
  2. From the Enterprise category, select one of the EJB module project templates.
  3. Follow the steps in the rest of the wizard. For more information, see the following topics:
    • Enterprise Application Wizard
    • Existing Blueprints Application Wizard
    • Enterprise JavaBeans Module Wizard
    • Enterprise JavaBeans Module with Existing Sources Wizard
