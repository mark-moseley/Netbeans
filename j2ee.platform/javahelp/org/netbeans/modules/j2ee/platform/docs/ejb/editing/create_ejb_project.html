Creating an EJB Module Project

See Also

  1. Choose File  New Project (Ctrl-Shift-N) from the main window.
  2. From the Enterprise category, select one of the following project templates:
    • Enterprise Application. Lets you create an EJB module project as part of an empty enterprise application.
    • Existing Blueprints Application. Imports an existing enterprise application.
    • EJB Module. Creates an empty EJB module project.
    • EJB Module with Existing Sources. Imports an existing EJB module into a standard project.
    • EJB Module with Existing Ant Script. Imports an existing EJB module into a free-form project.
  3. Follow the steps in the rest of the wizard.
Notes:
  • When importing an existing EJB module, the module can have any folder structure. When importing an existing BluePrints application, the project's folder structure must adhere to the J2EE Blueprints recomendations. For more information, see:

    .

  • The IDE only supports J2EE 1.4 EJB modules and automatically updates all J2EE 1.3 EJB modules to J2EE 1.4. The J2EE 1.4 specification is fully backwards-compatible, so upgrading the specification level does not change the behaviour of your code.
