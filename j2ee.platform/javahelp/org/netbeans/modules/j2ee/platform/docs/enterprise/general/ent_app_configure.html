Editing an Enterprise Application's Deployment Descriptors

See Also

An enterprise application includes two deployment descriptor files:

  • application.xml
    This J2EE-standard descriptor describes the application, which components it uses, how the components relate to each other, and which resources they use. Entries in this file are automatically generated when you add a module to the enterprise Application.
  • sun-application.xml
    This optional descriptor file defines configuration options that are specific to the Sun Java System Application Server.

Both deployment descriptors are XML files that are stored in the enterprise application's META-INF folder.

To edit the deployment descriptors of an existing enterprise application:

  1. In the Projects window, locate the project and expand its Configuration Files node. Alternatively, in the Files window, expand the project's build/META-INF subfolder after you have built the project.
  2. Double-click the node to open the file in the Source Editor window.
  3. Edit the elements. You can use code completion features to ensure that the elements are syntactically correct.
  4. (Optional) Right-click in the Source Editor and choose Validate from the contextual menu to verify that the document follows the XML schema for web deployment descriptors.

Troubleshooting

If the port number of the server changes, the changes to the deployment descriptor do not take effect until you restart the web application.
