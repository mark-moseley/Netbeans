Registering a Server

See Also

Before you can deploy a web application, JSP, servlet, or EJB module, the server to which you are going to deploy needs to be registered with the IDE. By default, only the bundled Tomcat Web Server is registered with the IDE. You must register all other servers yourself.

When you register a server with the IDE, you make its libraries available for production, or deployment, or both. If you deploy your applications to a remote instance of the Sun Java System Application Server, its libraries are available at runtime. However, during development, you might need a local instance of this server. For example, when you register the Sun Java System Application Server with the IDE, you can use its libraries for the following development activities:

  • Using the web service compiler to compile web services and clients
  • Creating a new web application with JavaServer Faces (JSF) support
  • Adding JSF support to an existing web application

You can have multiple instances of the Sun Java System Application Server registered with the IDE. Once you have registered your local instance, you can register remote instances, but you always have to specify the local folder that contains the local installation of the Sun Java System Application Server.

To register an external Tomcat Web Server or a Sun Java System Application Server:

  1. Go to Tools  Server Manager.
  2. In the Server Manager, click Add Server.

    3. The Add Server wizard appears and displays the types of servers that are compatible with the IDE.

  3. In the dialog box, select the type of server you want to register and click Next.
  4. Specify server-specific information in the panels that follow and click Finish.
  Alternatively, right-click the Server Registry node in the Runtime window. Then choose Add Server from the contextual menu and use the Add Server wizard to register the server.
