WSDL Binding Types: FTP

See Also

When creating or editing a WSDL file, you may be prompted to select a binding type and a binding subtype. A binding contains protocol and data format information for the operations and messages of a port type.

If you select the FTP binding type, then you must select one of the following binding subtypes. Both subtypes provide basic FTP functionality, such as a directory from which to read or write files, filename pattern matching, and message correlation.

  • FTP Message. Select this binding subtype if you plan to rely on default values.
  • FTP Transport. Select this binding subtype if you want to add customizations such as specifying custom directories on the FTP server and specifying pre-transfer and post-transfer operations.
