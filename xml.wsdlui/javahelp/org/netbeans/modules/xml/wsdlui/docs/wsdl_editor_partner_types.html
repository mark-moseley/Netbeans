Configuring Partner Link Types Using the Partner View

See Also

The PartnerLinkTypes box contains one configuration box for each partner link type in the WSDL file.

A partner link type can have one or two roles. The first role appears in the right side of the configuration box. The second role appears in the left side of the configuration box.

The role's port type appears below the role. A port type has one or more operations, which are illustrated in the space below the port types. A solid horizontal arrow indicates an input message. A dashed horizontal arrow indicates an output message. If the category of operation is Request-Response or Solicit-Response, then you can optionally add a fault.

  • The configuration box resembles a sequence diagram in Unified Modeling Language (UML).

To create a partner link type:

  1. Click Add PartnerLinkType.
    The configuration box for the partner link type appears. The default role appears in the right side of the configuration box.
  2. In the right side of the configuration box, double-click Enter or Select an Existing Port Type and specify the port type that the service will use to receive messages. To create a new port type, enter the name. To use an existing port type, select the port type from the drop-down menu.
  3. If you created a new port type, then drag one or more operations from the Palette window onto the vertical line that extends below the port type.
  4. (Optional) To add a second role, do the following:
    1. In the left side of the configuration box, double-click [External] and enter the role name.
    2. In the left side of the configuration box, double-click Enter or Select an Existing Port Type and specify the port type that the external service will use to receive messages.

To change the name of a partner link type:

  1. Double-click the partner link type name.
  2. Change the name.
  3. Press Enter.

To change the name of a role:

  1. Double-click the role name.
  2. Change the name.
  3. Press Enter.

To change the port type of a role:

  1. Double-click the port type.
  2. Change the port type. To create a new port type, enter the name. To use an existing port type, select the port type from the drop-down menu.

To add a fault to a request-response or solicit-response operation:

  1. Right-click the operation name and choose Add > Fault.
    The fault appears with a blank message.
  2. Specify the fault message.

To delete a role:

  1. Select the role.
  2. Press Delete.

To remove a partner link type:

  1. Select the configuration box for the partner link type.
  2. Click Remove PartnerLinkType.
