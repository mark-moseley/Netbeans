Writing a JMX MBean JUnit test.

JMX JUnit tests are full JUnit tests that handle the JMX MBean deployment tasks in order for the developper to focus on testing MBean attributes and operations.

You have to go through the following 5 steps in order to achieve MBean unit testing :

1) Generate the MBean JUnit test

JMX JUnit tests can be generated from :
  • The JMX MBean wizard. During the MBean creation, a JUnit test case can be generated. Using this wizard allows to generate test cases for any kind of MBeans (mainly Standard or Dynamic). If yo uwant to generate JUnit tests for new MBeans, you should use this wizard.
  • The JMX MBean JUnit Test wizard. This wizard allows you to generate a JMX JUnit test for an existing MBean. It does its best job when generating a test for a Standard MBean. When generating a test for a Dynamic MBean, it will only handle the MBean deployment tasks. You will have to add manualy the test cases for each attribute / operation you whish to test. If a JUnit test already exist for this MBean class (we look for <mbean package>.<MBeanClassName>Test file name in the test area), no test can be generated.
  • The Create JUnit Test... contextual action accessible from management menu. When an MBean is selected, the action becomes enabled (If no test already exists). This action has the same behavior as the previous wizard.

Selecting an MBean constructor

The MBean that you whish to test can have multiple constructors. When you will generate a JUnit test from JMX MBean JUnit Test wizard or Create JUnit Test... action, you will be asked to select a constructor from the list of MBean constructors.
In the case,  of a JUnit test generated from the JMX MBean wizard, the generated MBean default constructor is used.

2) Updating MBean construction in the JUnit generated test

If your MBean under test needs some parameters at construction time, you can provide these parameters thanks to the createMBean method.
Go to the createMBean method and update the MBean constructor call. This is not needed if your MBean construtor is a void one.

In the following example, the MBean class MYClass expects an Object parameter.

public Object createMBean() {
    //TODO update MYClass(Object o) constructor parameters with valid values.
    return new MYClass(null);
 }

3) Adding test logic to an MBean attribute test

For each attribute (if any) a test case is generated. The test case contains a call to get the attribute.
After the attribute has been get, a call to fail is generated. You have to replace the call to fail method with your test logic.
The following code example test Toto attribute.

/**
* Test of Toto Attribute
*/
public void testTotoAttribute() throws Exception {
    System.out.println("testTotoAttribute");
   
     // Get JMX MBean Attribute.
     Boolean val = (Boolean) getAttribute("Toto");
       
    // TODO add your test code below by replacing the default call to fail.
    fail("The test case is empty.");
}

4) Adding test logic to an MBean operation test

For each operation (if any) a test case is generated. The test case contains :
  • The operation signature creation.
  • The operation parameters creation that you have to update with valid values. Some default values are generated.
  • The operation invocation.
  • A call to fail method that you have to replace with your test logic.

The following code example is a test for operation addAll(Integer, java.util.Collection)
/**
* Test of addAll Operation
*/
public void testAddAllOperation() throws Exception {
    System.out.println("testaddAllOperation");

    // Operation signature.
    String[] signature = new String[] {
        java.lang.Integer.TYPE.getName(),
        java.util.Collection.class.getName()
    };
       
    // TODO update with your operation values.
    Object[] params = new Object[] {
        0,
        null
    };
       
    // Invoke JMX MBean Operation.
    Boolean val = (Boolean) invoke("addAll",params,signature);
       
    // TODO add your test code below by replacing the default call to fail.
    fail("The test case is empty.");
}

5) Running the JUnit test

Like any other JUnit test, select your MBean test file, right click then select Run File action.
