Registering Java objects as MBeans in a JMX Agent


Contextual (enabled only if an JMX Agent is selected) action located under the management menu. You can access the management menu from :
  • Top level menu
  • Project tree node contextual menu (right click on selected node)

Action applicability


This action  applies when a JMX Agent is selected. A JMX Agent class is a class generated from the JMX Agent Wizard.
The JMX Agent init method is updated with the code needed to :
  • Instantiate the java object. This instantiation is optional. The java object to wrap can be instantiated anywhere else in your code.
  • Wrap this java object in a javax.management.StandardMBean JMX class instance.
  • Name (Provide an MBean ObjectName)
  • Register (Call MBeanServer register method)
You can call this action multiple time to register multiple Java objects.

Java classes that you can wrap

Not all Java classes are good candidate for automatic exportation as MBean. Some checks are applied by JMX. A class that can be wrapped is :
  • A class that implements at least 1 interface. One of the implemented interface will be exported as the management interface.
  • A class that is compliant with JMX Standard MBean design pattern.

Using the Instantiate and Register MBean... action

  1. From the management menu select Instantiate and Register MBean...
  2. A popup windows is displayed.
  3. Select Register Java Object Wrapped in a StandardMBean checkbox
  4. You can click on Browse... button to select a Java class from the current project. You can also enter a Java class name directly in the textfield.
  5. Once you entered a valid Java class, ObjectName textfield becomes active. A default ObjectName is provided. The ObjectName default computation is :
    • Domain name is derivated from MBean package name. The package name is inverted (eg: com.foo.bar ==> bar.foo.com). If the package name is the default package name, the JMX default domain is used.
    • A single key named type having the class name as its value. (eg: class MyClass ==> type=MyClass)
  6. You are free to add any key/value pairs. The default ObjectName computation is well suited to name "singleton" MBean. If you want to register more than 1 Java class of the same type, you should add at least a key to avoid naming conflict.
  7. You can now select a Class constructor from the Construtor dropdown list. If you don't want to generate the Java object instantiation, select "Object Instantiation Not Generated" item.
  8. Click OK when you are done.
  9. The JMX Agent source code is loaded in the editor.
  10. Goto the edit method to visualize generated code.
  11. If you don't need to update the generated code (no object reference to provide nor no parameters to provide to the Java class constructor), you can now compile, run and visualize MBeans living in your Agent. Just click on Run/Run Main Project with JConsole..., or on its associated toolbar icon. Compilation, execution and visualization will be processed automatically.

Updating the generated code


In the Agent source file go to the init() method

  • If you selected "Object Instantiation Not Generated", you need to provide your own class instance. In the following extract, a class MyClass implementing an interface AnInterfaceI is wrapped in a StandardMBean. Just replace the null parameter with your own object reference.
                getMBeanServer().registerMBean(
                    //TODO replace first parameter by your own MyClass object.
                    new javax.management.StandardMBean(null, I.class),
                    new ObjectName(":type=Test2"));

  • If you selected a construtor that needs some parameters, you need to update the generated code with such parameters. Provide needed parameters to the wrapped Java class constructor. In the following extract, a class MyClass implementing an interface AnInterfaceI is wrapped in a StandardMBean. The MyClass constructor needs a String at construction time. Just replace the null parameter with a String implementation.

                //TODO update MyClass(String) constructor parameters with valid values.
                getMBeanServer().registerMBean(
                    new javax.management.StandardMBean(
                        new MyClass(null, AnInterfaceI.class),
                    new ObjectName(":type=MyClass"));
