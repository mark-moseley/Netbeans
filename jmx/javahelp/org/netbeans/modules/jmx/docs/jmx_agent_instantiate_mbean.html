Registering MBeans in a JMX Agent


This is performed by a contextual action (enabled only if an JMX Agent is selected) located under the Management menu. You can access the management menu from :
  • The top level menu
  • The the contextual menu of the project tree node (right click on selected node)

Applicability of the Action


This action  applies when a JMX Agent is selected. A JMX Agent class is a class generated from the JMX Agent Wizard.
The JMX Agent init method is updated with the code needed to :
  • Instantiate (Call an MBean constructor)
  • Name (Provide an MBean ObjectName)
  • Register (Call the MBeanServer register method)
an MBean within the platform MBeanServer. You can call this action multiple times to register multiple MBeans.

Using the Instantiate and Register MBean Action

  1. From the Management menu, select Instantiate and Register MBean...
  2. A popup window is displayed.
  3. Register Existing MBean is already selected.
  4. You can click on the Browse... button to select an MBean from the current project. You can also enter an Mbean class name directly in the textfield.
  5. Once you have entered a valid MBean class, the ObjectName textfield becomes active. A default ObjectName is provided. The computation of the default ObjectName is as follows:
    • The Domain name is derivated from MBean package name. The package name is inverted (eg: com.foo.bar ==> bar.foo.com). If the package name is the default package name, the JMX default domain is used.
    • A single key named type, having the class name as its value (eg: class MyClass ==> type=MyClass).
  6. You are free to add any key/value pairs. The default ObjectName computation is well suited to naming "singleton" MBeans. If you want to register more than one MBean of the same type, you should add at least a key to avoid naming conflict.
  7. You can now select an MBean constructor.
  8. Click OK when you have finished.
  9. The JMX Agent source code is loaded in the editor.
  10. Go to the edit method to visualize generated code.
  11. If you don't need to update the generated code (no parameters to provide to the MBean constructor), you can now compile, run and visualize the MBeans living in your Agent. Just click on Run/Run Main Project with JConsole..., or on its associated toolbar icon. Compilation, execution and visualization will be processed automatically.

Updating the generated code

If your MBean needs some parameters at construction time, you need to update the generated code with these parameters.

  1. In the Agent source file, go to the init() method
  2. Provide the required parameters to the MBean constructor. In the following extract, the MBean constructor needs a String.

//TODO update Status(String) constructor parameters with valid values.
getMBeanServer().registerMBean(
    new Status("OK message"),
    new ObjectName("foo.bar.com:type=Status"));
