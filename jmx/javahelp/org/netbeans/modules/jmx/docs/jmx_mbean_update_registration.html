Making an MBean implements javax.management.MBeanRegistration interface


Contextual (enabled only if an MBean is selected) action located under the management menu. You can access the management menu from :
  • Top level menu
  • Project tree node contextual menu (right click on selected node)

Action applicability


This action  applies to any MBean type. If an MBean already implements this interface, the action is disabled.
This action applies to any MBean. You don't need to have them generated from the MBean wizard.
MBeans imported from other context can be updated.

Using the Implement MBeanRegistration Interface... action


  1. From the management menu select Implement MBeanRegistration Interface...
  2. A windows is displayed with a message listing the set of methods that will be added to the current MBean
    • preRegister : Called before the MBean is registered in the MBeanServer
    • postRegister : Called after the MBean is registered in the MBeanServer
    • preDeregister : Called before the MBean is unregistered in the MBeanServer
    • postDeregister : Called after the MBean is unregistered in the MBeanServer
  3. You have the ability to generate private fields in order to keep some references to preRegister method parameters. These parameters are :
    • The MBean ObjectName. If you are interested in the name the MBean is given, you should keep a reference to this parameter.
    • The MBeanServer in which the MBean is registered. If you want your MBean to make JMX call to other MBeans, you should keep this reference.
  4. Click OK when you are finished
  5. MBeanRegistration methods are added at the end of the MBean class.
  6. The updated MBean class file is automaticaly opened in the editor.
