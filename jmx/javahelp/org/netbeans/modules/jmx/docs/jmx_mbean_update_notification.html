Implementing the javax.management.NotificationEmitter Interface


NotificationEmitter is implemented using a contextual action (enabled only if an MBean is selected) located under the Management menu. You can access the management menu from :
  • The contextual menu for the project tree node (right click on selected node)

Applicability of the Action


This action  applies to any MBean type. If an MBean already implements this interface, the action is disabled.
This action applies to any MBean. You don't need to have generated them using the MBean wizard.
MBeans imported from other contexts can be updated.

Using the Implement NotificationEmitter Interface Action


  1. From the JMX menu select Implement NotificationEmitter Interface...
  2. The wizard can generate an implementation of the NotificationEmitter interface based on a NotificationBroadcasterSupport delegation. Some code to handle the notification sequence number can also be generated. Check the associated checkboxes to have this code generated.
  3. If you want your MBean to define notifications, click Add Notification.
  4. In the Notifications Table, first row, type in the following:
    1. In the column Notification Class you can select a standard JMX notification class or type in your own Notification class. If you select AttributeChangeNotification class, the Notification type will be automatically set to AttributeChangeNotification.ATTRIBUTE_CHANGE.
    2. In the column Description, type in a notification description (eg: "Send a notification when the Status attribute is changed")
    3. In the column Notification type,  click the Edit button.
      1. Click Add Type
      2. In the column Notification, you can change the default type to your own type (eg : com.foo.event.BIRTHDAY)
  5. Click OK when you have finished.
  6. NotificationEmitter methods are added at the end of the MBean class.
  7. The updated MBean class file is automatically opened in the editor.
