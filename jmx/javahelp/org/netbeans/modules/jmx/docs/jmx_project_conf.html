Java Application Project Configuration

Management actions are applied to Java Application Project type only.

Configuration steps :
  1. You have to set a Java Application project as your main projet.
  2. Set properly your Runnable Class in the project properties. This can be done automatically when generating a JMX Agent.
  3. Check existing management related VM Options (if any). Be sure to remove these options from the project properties menu. It can conflict with these actions.
  4. FAT File System issue. Run Main Project with JConsole and Debug Main Project with JConsole actions are not working on FAT (see JConsole FAQ for more details on the issue http://java.sun.com/j2se/1.5.0/docs/guide/management/faq.html)
    • Use Run > Remote Management > Run Main Project with Remote Management or Debug Main Project with Remote Management actions instead
    • Or, to workaround this issue, add -XX:+PerfBypassFileSystemCheck to the project VM Options (Project Properties > Run > VM Options textfield). DON'T use this option in real environment. It is an unsupported option that workaround the problem BUT makes the management access unsecure.
