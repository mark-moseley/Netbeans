About Actions in the Swing Application Framework

See Also

The Swing Application Framework provides support to help you define and manage actions.

You define actions with the &Action annotation and the framework encapsulates all such defined actions with its ActionMap class. In addition, you can configure actions to run asynchronously.

In the IDE, you can use Set Action dialog box to generate code for actions. In the Set Action dialog box, you can do the following:

  • Set a component to use an action.
  • Create skeleton actions.
  • Configure user interface properties for the action, such the name to appear on the visual component, tool tip text, and keyboard shortcuts.
  • Create a Boolean property to determine whether the action is enabled.
  • Create a Boolean property to determine whether the action is selected. (This only applies to non-menu components that have a selected property, such as radio buttons.)
  • Set an action to run as a background task.
  • Configure an action to block other actions until it completes.

To open the Set Action dialog box, do one of the following things:

  • Right-click a component and choose Set Action.
  • Select a component and click the ellipsis (...) button next to its action property in the Properties window.
  • When you assign an action to a component, that component's display name, tool tip, and accelerator are taken from what you enter in the Set Action dialog box. If you have previously set these properties for the component, they will be overridden by what is specified in the Set Action dialog box.
