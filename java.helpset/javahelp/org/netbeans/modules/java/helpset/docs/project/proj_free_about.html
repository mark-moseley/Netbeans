About Free-Form Projects

See Also 

The IDE also contains free-form project templates for the various categories of projects (such as General Java, Web, and Enterprise). Free-form projects do not produce an Ant script or hold project metadata in another form. Instead, these projects rely on your existing Ant script to provide instructions for handling your project source.

The standard distribution of the IDE contains the following free-form templates:

  •  Java Project with Existing Ant Script. A free-form project containing one or more Java source roots.
  •  Web Application with Existing Ant Script. A free-form project containing a web application and optionally other Java source roots.
  •   EJB Module with Existing Ant Script. A free-form project containing an EJB module and optionally other Java source roots.
  • Depending on what modules you have installed, your IDE may contain additional templates. Consult the help for your plug-in modules for more information.

Source Folders

Free-form Java projects can contain multiple source folders. The classpath relationships and handling instructions for your source directories must be handled in your Ant script.

Project Settings

In the project's properties, you declare the source folders, classpath, and output files for your project. Each source root can have a unique classpath and output file. These settings do not affect the actual contents or classpath of your application. The settings only tell the IDE how to handle the code. For example, the classpath settings tell the IDE which classes to make available for code completion.

IDE Commands and Ant Targets

In free-form projects, the IDE relies on your Ant script to provide targets for all IDE actions, such as running, debugging, and generating Javadoc. If your Ant script does not contain targets for these commands, the commands are disabled.

You can easily write debug targets in your Ant script or in a supplementary Ant script. You can also add a shortcut to any of your Ant script's targets to the contextual menu of your project's node in the Projects window.
