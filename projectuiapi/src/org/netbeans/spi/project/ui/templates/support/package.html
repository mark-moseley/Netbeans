Support for providing own templates visible in New File... action.

Each project can declare own template and exposing it in New File... action to allow users to create new file types supported by this project. The project can provide own iterator to a project-specific customization.

The client project has to declare its templates in the module's layer file in folder Templates. The declaration looks like <folder name="Templates">
    <folder name="JSP_Servlet">
      <file name="JSP.jsp" url="templates/JSP.template">
        <attr name="SystemFileSystem.localizingBundle" stringvalue="org.netbeans.modules.web.core.Bundle"/>
        <attr name="template" boolvalue="true"/>
        <attr name="instantiatingWizardURL" urlvalue="nbresloc:/org/netbeans/modules/web/core/resources/JSP.html"/>
        <attr name="instantiatingIterator" methodvalue="org.netbeans.modules.web.dd.wizards.PageIterator.createJspIterator"/>
        <attr name="templateCategory" stringvalue="web-types"/>
      </file>
   </folder>
</folder>

The subfolder of Templates folder specify the folder in which will be this templates present in templates chooser invoked from New File... action. In this subfolder is declared file which represents one template and its attributes: The file tag has to have a unique name to recognize in the SystemFileSystem and the url property which links to file pattern of the declared template.

The mandatory file's attributes are:
SystemFileSystem.localizingBundle specifies the ResourceBundle which contains the localized display name of template,
template must be true otherwise will be ignored.

The optionally attributes are:
instantiatingWizardURL links to a resource with a template's description,
instantiatingIterator specifies a instance of org.openide.InstantiatingIterator which instantiate new file in specified target folder,

    Note: a client project can declare own custom iterator with some project-specific customization,
    it's recommended use the standardized target chooser exposed in {@link org.netbeans.spi.project.ui.templates.support.Templates.createSimpleTargetChooser}, 
    a similar target chooser offers {@link JavaTemplates.createPackageChooser}
    for Java-oriented projects.
    The client-project also can use a InstantiatingIterator published by other module,
    i.e. org.netbeans.spi.java.project.support.ui.templates.JavaTemplates.createJavaTemplateIterator
    useful for Java-oriented templates in case of no needs to own customization.
    The last possibility is declaration no iterator, in this case will be used a generic iterator
    useful for simple file types without any specific needs, i.e. properties file.

template-category files the template to some category. The template's categories helps to filter the templates according to type of project. The project provides the project's RecommendedTemplates.
