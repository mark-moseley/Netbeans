Edit security-mechanism Profile Dialog Box

See Also

The Edit security-mechanism Profile dialog box lets you configure message-level security for an instance of the Sun Java System Access Manager server.

You can configure the following settings:

  • Sign Response. Specifies whether responses should be signed.
  • Existing Certificate Settings.
    • Use Default Key Store. Lets you use the default key store. When selected, the other fields in this area are disabled.
    • Keystore Location. Specifies the location of the keystore file that contains the key alias.
    • Keystore Password. Specifies the password needed to access the keystore file.
    • Key Alias. Specifies the key alias for the entry in the keystore. Each entry in a keystore file is identified by an alias string.
    • Key Alias Password. Specifies the password needed to access the key alias for the entry in the keystore.
  • Username Token Profile Info. These settings appear only when the UserNameToken security mechanism is in effect.
    For each web service client that is granted access to the server, specifies the Username and Password for the token profile.
    • The values in each row of this table must match the values defined for the individual web service clients in the Security tab of the Web Service Attributes Editor.
