Enabling the Liberty Security Mechanism for Web Service Providers

See Also

You use the Security tab of the Web Service Attributes Editor to enable and configure the Liberty security mechanism for web service providers.

To enable the Liberty security mechanism for a web service provider:

  1. In the Projects window, expand the project node for the project where your web service provider is defined.
  2. Expand the Web Services node, right-click the web-service node, and choose Edit Web Service Attributes from the pop-up menu.
    The Web Service Attributes Editor opens with the Security tab open.
  3. Select the Enable Message-Level Security checkbox.
  4. Under Security Mechanism, from the Request drop-down list, select one of the Liberty tokens.
    For the Liberty security mechanism, the web service provider must be registered with Access Manager's Discovery Service.
  5. Click OK.
  • The certificate settings at the domain level for this token profile will apply for the server you selected. To change the global settings, follow the instructions in Configuring Security Mechanisms for Sun Java System Access Manager Instances.
  • For the Liberty security mechanism, if you want to use a custom keystore, you must perform the steps described in Using a Custom Keystore for the Liberty Token Profile before deploying or running the project.
