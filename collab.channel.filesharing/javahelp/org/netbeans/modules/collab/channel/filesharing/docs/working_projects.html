Working with Shared Projects and Project Files

See Also

You can perform actions on projects and project files that share. Which actions you can perform depend on whether you are a local or remote user -- that is, whether you are the original owner of the shared files, and whether the actions you want to perform are on projects or on project files.

Projects and files that are shared are subject to the following rules::

  • File owners can run compilation and debugging actions on original files in their local file systems.
  • Remote users who are sharing Projects or Project Files can see the output of a local compilation in the output window. Remote users can perform remote compilation, and run actions on Shared projects.To do so, they must invoke the actions on the shared Project node
  • When the original owner shares a Project, src files are shared. If you want to share nbproject and build.xml files you must drag and drop or copy these files individually from the IDE's Files Tab into the File Sharing Channel.
  • You can share all files in a Project. Files that are not part of a Project will reside in the Shared Non Project Files folder in the File Sharing Channel.
  • You can right-click on the New operation from the root node in the File Sharing Channel to create new shared files that are not affiliated with existing Projects in the IDE.

To build projects that are shared (local user):

  1. Right-click a project node in either the original file system or in the File Sharing Channel.
  2. Select the action you want to perform.
    You can run any project-related action, either locally or on your file system or from the File Sharing Channel.

To compile files that are shared (local user):

  1. Right-click a source file in the original file system to perform any action related to compiling a file.
  2. Select the action you want to perform.
    Local and remote users will see the output of this project.
    Local users cannot compile source files in the File Sharing Channel.

To build projects that are shared (remote user):

  1. Right-click on a project node in the File Sharing Channel.
  2. Select an action you want to perform. (Local and remote users will see the output of this project.)

To work with shared files if you are the remote user:

  • Double-click a shared file in your File Sharing Channel to open or edit the file.

To rename shared files if you are a local user:

  1. Select a project node or a source node on a project.
  2. Drag the project node to the File Sharing Channel.
  3. On the Project Tab, select the file you want to rename.
  4. Right-click that file and select Refactor.
  5. Select Rename and rename the file to a name of your choosing.

To delete a shared project file if you are a local user:

  1. In the Projects window, select the file you want to delete.
  2. Right-click and select Delete.
    The file is deleted from the Projects window, from the File Sharing Channel, and from the remote user's File Sharing Channel.

To unshare a file if you are a local user:

  1. Select a project node or source node in the Projects window.
  2. Drag the project node to the File Sharing Channel.
  3. In the File Sharing Channel, select the file you want to unshare.
  4. Right-click and select Stop Sharing.
