Sharing Projects

See Also

You can use the Developer Collaboration feature to share projects with other developers within a conversation. Projects are shared using the Shared Files collaboration channel visible within each conversation window. When you share projects, you share the project structure, and many, but not all, of the files in the original project. References in the original project structure are not kept when projects are shared.

  If the original owner of a file attempts to share the file with a second remote sharer, the original owner will receive a warning popup. Thus you cannot share files across different conversations.
When the file owner leaves a conversation, ownership of a project file will be transferred to a random participant.

You can share the following types of projects directly:

  • Java Application
  • J2EE Application
  • Web Application
  • EJB Application
  • Ant Application

When you share a project, most, but not all, Project files, folders and nodes are copied to the Shared Files collaboration Channel. The Libraries and Test Libraries Project nodes are not copied when you share Projects. The following Project files or folders are not copied when you share Projects:

  • build.xml
  • nbproject
  • build
  • dist

To share projects:

  • To share an existing project, drag or copy the project into the File Sharing channel of a conversation.
  • To share a new project, select a folder node in the File Sharing channel of a conversation and use the contextual menu or File > New to create the desired project.

The projects are sent to the other conversation participants automatically, except for project artifacts. Note that when you create new Projects in the File Sharing Channel, you create a new Project in your original filesystem.

  When other participants share projects, their files appear in your File Sharing channel a short time after a short delay. Each project shows the owner, or person who originally shared the project, after the project name.
Most project files shared within a conversation can be collaboratively edited in real time. In general, if the IDE can edit the content of a file within a project locally, it will be editable once shared. See Editing Shared Files for more information.

To delete a shared file or folder in a shared project:

  • Select the file or folder node of the project file and press Delete or use the contextual menu to remove the file or folder.
    Only project file owners may delete a shared project file.

To save changes made to a shared file:

  • Drag or copy the shared file within a shared project from the File Sharing channel back to a local filesystem.
  When a conversation is ended (all participants leave), all shared project files are discarded and are no longer accessible. However, backups of shared project files are kept for a time in your IDE user directory under the collab/shared_files folder. The number of conversation backups to keep is configurable in the File Sharing channel options under Tools > Options, in the Collaboration /Channels category.
