Reverting to the Default Configuration

See Also

There are two ways to revert edits made in the CASA Editor and restore previous configurations:

  1. You can revert to the default configuration for the composite application.
  2. You can revert a WSDL port to the configuration defined by a JBI module.

To revert to the default configuration of a composite application:

  1. In the Project window, right-click the composite application node and select Clean Application Configuration.
    All modifications to the application configuration are deleted and the CASA Editor is closed if it was previously open.

    Caution: If you perform this procedure, all modifications that you have made cannot be recovered.

To revert a WSDL port to the configuration defined by a JBI module:

  1. In the WSDL Ports section of the CASA Editor, delete the WSDL binding to which the JBI module connects.
  2. Right-click the JBI module and select Load WSDL Port.
    The default WSDL ports are added to the WSDL Port section of the CASA Editor.
  3. Manually connect the WSDL ports to the JBI module.
