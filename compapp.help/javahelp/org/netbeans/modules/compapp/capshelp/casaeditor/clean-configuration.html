Reverting to the Default Configuration

See Also

There are two ways to revert to an application's default configuration.

  • You can abandon all changes made to an application's configuration and revert to the default configuration.
  • You can revert an individual JBI module to its default configuration.

To revert to the default configuration of a composite application:

  1. In the Project window, select and expand the project node of the Composite Application project, right-click the Service Assembly node under the project node and select Clean.
    All modifications to the application configuration are deleted and the CASA Editor is closed if it was previously open.

    Caution: If you perform this procedure, all settings are returned to the default values, and any modifications that you have made cannot be recovered.

To revert an individual JBI module to its default configuration:

  1. In the WSDL Ports section of the CASA Editor, delete the WSDL binding to which the JBI module connects.
  2. Right-click the JBI module and select Load WSDL Port.
    The default WSDL ports are added to the WSDL Port section of the CASA Editor.
  3. Manually connect the WSDL ports to the JBI module.
