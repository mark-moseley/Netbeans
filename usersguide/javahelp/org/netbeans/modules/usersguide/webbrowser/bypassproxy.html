Accessing Local Files Through a Proxy

If you want to use the Update Center or access the Internet through the IDE but your system is behind a firewall or you use a proxy server, you need to configure the IDE with your proxy settings. When you set the IDE to use a proxy, however, your Internet browser may attempt to route all files through the proxy, including local files. This results in a File Not Found error when you try to open HTML files or applets in your external browser from the IDE.

To resolve this problem, add localhost and your machine name to the list of bypassed hosts in your browser's settings.

For Netscape, choose Edit  Preferences, expand the Advanced Node, and select Proxies. Then click the View button and add the information to the Exceptions field.

For Internet Explorer, choose Tools  Internet Options, click the Connections tab,

To set the bypassed hosts for Netscape Communicator:

  1. Open the Netscape browser and choose Edit  Preferences from its main window.
  2. Expand the Advanced Node and select Proxies.
  3. Click the View button next to Manual Proxy Configuration.
  4. Type localhost and your machine name in the Exceptions field.
  5. Click OK to apply the changes to your browser settings.

To set the bypassed hosts for Internet Explorer:

  1. Open Internet Explorer and choose Tools  Internet Options from its main window.
  2. Click the Connections tab and click the LAN Settings button.
  3. Click the Advanced button under Proxy Server.
  4. Type localhost and your machine name in the Exceptions field.
  5. Click OK to apply the changes to your browser settings.
