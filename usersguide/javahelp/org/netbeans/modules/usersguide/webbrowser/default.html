Changing the Default Web Browser

See Also

The IDE uses the IDE's default web browser to run a web application. You can set the default web browser to be one of the supported browsers, such as the supported versions of Mozilla, Netscape, and Internet Explorer, or to be the IDE's internal web browser. You can also configure the IDE to use other browser types.

  1. Choose Tools  Setup Wizard from the main menu.
  2. Select a browser from the Web Browsers combo box.

    3. You can choose from the following web browsers:

    • <System Default>. The browser that is registered as your operating system's default browser.
    • Swing Browser. A simple HTML browser based on a Swing component. The internal Swing HTML browser provides a higher level of integration with the IDE than an external browser. You can embed the Swing HTML browser into another IDE window or an MDI frame.
    • Any other browsers installed on your computer.

If you select a web browser in the Setup Wizard and the web browser still does not open, check that the IDE has the correct location of the browser executable.

To specify the location of the browser executable:

  1. Choose Tools  Options from the main menu.
  2. Expand IDE Configuration  Server and External Tools Settings  Web Browsers and select the browser type you are using.
  3. Open the property editor for Browser Executable property and set the full path to the executable in the Process field. You can click the ellipsis button (...) to browse to the executable.

Troubleshooting

If you get "file not found" errors when you execute a web application, see Accessing Local files Through a Proxy for possible solutions.

If the browser does not reflect changes in in a JSP file or HTML file, this might be because of the way the browser is caching pages. Check the browser's settings to ensure that the browser is reloading the page every time you access it.
