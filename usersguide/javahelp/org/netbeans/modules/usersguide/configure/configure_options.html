Options Window

To open the Options window:

  • Choose Tools Options.

You can use the Options window to change any of the IDE's configurable settings. The left pane of the window contains a tree view of all of the configurable items in the IDE. The Origin of Setting (<<) column displays the icon for settings that have a configuration specific to the current project. The right pane displays a property sheet, which contains the names and values of the settings for the item selected in the left pane. For some types of settings, there multiple tabs for distinct categories of properties (such as Sorting and Execution).

To edit a setting, click the setting's value in the property sheet.

  • If the value is a string, you can edit the value inline and press Enter to save the change.
  • If a drop-down arrow appears when you click the value, you can select the arrow and choose the value from the combo box.
  • If an ellipsis (...) button appears, you can click the ellipsis button to open a property editor.

The IDE can store settings at three levels:

  • Default - holds the default settings that come with the IDE installation. You can not change the default settings, but you can override the default settings at the user or project level. If you modify a setting that is only defined in the default level, a definition with the modified values you provide is moved to the User level.
  • User - like default settings, settings defined at the User level apply to all of your projects. Settings defined at the User level override settings at the Default level.
  • Project - settings configured at the Project level only apply to the project in which they are configured. Any settings configured at the Project level override user settings and default settings.

If the cell in the Origin of Setting (<<) column for the node you are configuring is empty, any changes that you make in the property sheet are applied at the User level. If the cell in the Origin of Setting (<<) column for the node you are configuring displays the icon, any changes that you make in the property sheet are applied only to the current project.

You can see a table view of how the settings are stored by clicking the Origin of Setting (<<) cell. There are columns for project, user, and default settings. Each row represents a group of settings.

To configure a setting for the current project only:

  1. Choose Tools | Options.
  2. In the top row of cells, click the Origin of Setting (<<) cell to display the levels for the settings.
  3. Navigate to and select the node for the settings you want to configure.
  4. Click the cell in the Projects column for the settings node you have selected and choose Define Here.

    The icon appears in the Origin of Setting (<<) column.

  5. In the property sheet, configure the desired settings.

To revert from a project setting to a user setting:

  1. Choose Tools | Options.
  2. In the top row of cells, click the Origin of Setting (<<) cell to display the levels for the settings.
  3. Navigate to and select the node for the settings you want to configure.
  4. Click the cell in the Default column for the settings node you have selected and choose Revert Definition.

    You are then prompted to make sure you want to remove that settings node's configuration for the current project. If you click Yes, the project reverts to using the setting stored at the User level.

To revert to the default setting:

  1. Choose Tools | Options.
  2. In the top row of cells, click the Origin of Setting (<<) cell to display the levels for the settings.
  3. Navigate to and select the node for the settings you want to configure.
  4. Click the cell in the Default column for the settings node you have selected and choose Revert Definition.

    You are then prompted to make sure you want to remove that settings node's configuration for the current project. If you click Yes, the project reverts to using the setting stored at the Default level.
