IDE XML Tasks: Quick Reference

See Also 

The following table describes common XML editing tasks.

To perform this task Follow these steps
Create a new XML document.
  1. In the Versioning window, right-click the VCS working directory's top level node and choose CVS  Checkout.
  2. Specify the modules you wish to check out.
    • To check out the entire repository select All.
    • To check out a specific directory, click the Select button to choose from a list of all of the modules in the repository.
  3. Click Finish to check out the files.
Open an XML document in the Source Editor.
  1. Right-click the file's node that you wish to update.
  2. Choose CVS  Update.
Check an XML document.
  1. Right-click the appropriate revision node in the Versioning window.
  2. Choose Diff Graphical from the contextual menu.
Validate an XML document.
  1. Ensure that your local copies of the files are up-to-date by right-clicking and choosing CVS  Update before committing your changes.
  2. Right-click the files or directories you wish to commit and choose CVS  Commit.
Create a new CSS.
  1. In the Versioning window, expand the file's node to reveal the subnodes for each revision and branch.
  2. Right-click the revision you wish to merge into your working copy and choose Merge With Revision.
Generate a CSS
  1. Choose Versioning Versioning Manager from the main window.
  2. Click Add. The New Wizard appears allowing you to configure the necessary versioning settings.
  3. Select the CVS profile from the Version Control System Profile combo box.
Create a new DTD
  1. Right-click the locally modified file or folder and choose CVS  Diff Textual.
  2. In the Output dialog, ensure that the output shows the correct differences.
  3. Right-click in the output pane and choose Save To File from the contextual menu.

    A patch file is created containing the differences between the source code file versions.
Generate a DOM Tree Scanner.
  1. Choose Versioning Versioning Manager from the main window.
  2. Click Add. The New Wizard appears allowing you to configure the necessary versioning settings.
  3. Select the CVS profile from the Version Control System Profile combo box.
Generate a SAX document handler
  1. Choose Versioning Versioning Manager from the main window.
  2. Click Add. The New Wizard appears allowing you to configure the necessary versioning settings.
  3. Select the CVS profile from the Version Control System Profile combo box.
Create an XML schema
  1. Choose Versioning  Versioning Manager.
  2. Select the appropriate VCS node and click Edit.
  3. Edit the necessary VCS settings in the working directory's Customizer.
Configure global XML settings.
  1. Choose Versioning  Versioning Manager.
  2. Select the appropriate VCS node and click Edit.
  3. Edit the necessary VCS settings in the working directory's Customizer.
Configure XML and DTD Editor Properties.
  1. Choose Tools  Options, expand the Source Creation and Management node, then select the Version Control Settings node.
  2. Edit the desired properties in the Option window's right pane.
