XML Tools Cookies APIs

Currently we propose two common cookies.

CheckXMLCookie vs. ValidateXMLCookie

Both cookies are supported because it mimics users requests for separating fast syntax level check actions and accurate semantics level actions.

No UI Interaction Rule

We are proposing here to use No UI intaraction rule contrain for most cookie interfaces. It means that a cookie specification should state that implementation cannot change UI state. Rather the cookie interface should provide a UI hook that can be plugged in by a cookie client (that is responsible for UI interaction).
