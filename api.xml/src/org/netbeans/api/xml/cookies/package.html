XML Tools Cookies APIs

Currently we propose two common cookies. Once a MDR will be ready we can add model access APIs if not covered by MDR itself.

No UI Interaction Rule

We are proposing here to use No UI inetraction rule for all cookie interfaces. It means that a cookie specification should state that implementation cannot change state of UI. Rather it should provide a UI hook that can be plugged in by a cookie user (that is responsible for UI interaction).
