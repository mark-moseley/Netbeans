Editing and Replaying Requests in the HTTP Monitor

You can replay HTTP requests associated with both the Current Records and Saved Records subcategories of the All Records tree view. The executor resends the original HTTP request.

Before resending requests, verify that the current web module is being executed on the web server.

To resend an HTTP Monitor data record to the server:

  1. Right-click the monitor data record to be replayed.
  2. Choose Replay from the contextual menu. The page resulting from the request appears in the browser you chose.

Note that if a servlet or JSP page reads data directly from the ServletInputStream as opposed to using methods from the getParameter family, requests to that resource will not replay correctly.

To edit and then resend a monitor data record:

  1. Right-click the monitor data record to be edited.
  2. Choose Edit and Replay from the contextual menu. The Edit and Replay dialog box appears. It enables you to modify query parameters, request information, or headers before sending the request. You can also send the request to a different server.
  3. Edit information about the request.
  4. Click Send HTTP request.

When editing and resending a record, there is a variety of information which you can edit. The sections below describe the information which can be edited in HTTP Monitor's Edit and replay dialog.

To edit query parameters:

  1. Click the Query tab, and then click the Add parameter button. The Add Parameter dialog box appears.
  2. Type in a name and value, and click OK. The query parameters appear in the Edit and Replay dialog box.
  3. To delete a query parameter, select the parameter, and then click Delete parameters. Use the Shift or Control key in conjunction with Delete parameters to delete multiple parameters.

To edit request data:

  1. To edit a request parameter, such as Request URI or Protocol, click the Request tab. Click the ellipsis button next to the desired request parameter. The Edit Value dialog box appears.
  2. Type in a value, and click OK. The edited request parameter values appear in the Edit and Replay dialog box.
  3. To change a request method, select the desired method from the combo box in the Method field. For instance, change GET to POST.

To edit cookies:

  1. Click the Cookies tab, then click the ellipses button next to the appropriate cookie. The Edit Cookie dialog box appears.
  2. Edit the cookie name and value, and click OK. The edited cookie appears in the Edit and Replay dialog box.
  3. To delete a cookie, select the cookie and then click Delete cookies. Use the shift or Control key in conjunction with Delete cookies to delete multiple cookies.

To edit request headers:

  1. To add a header, click the Headers tab and then click the Add header button. The Add Header dialog box appears.
  2. Type in a name and value and click OK.
  3. To delete a header, select it, and then click Delete headers. A confirmation dialog appears. Use the Shift or Control key in conjunction with the Delete headers button to delete multiple headers.
  4. To edit a header parameter such as Accept, Connection, Host, or user-Agent, click the ellipsis button next to the desired parameter. The Edit header dialog box appears.
  5. Type in the new name and value and click OK. The new information is reflected in the Edit and replay dialog box.

To send a request to a different server:

  1. To edit server information, such as hostname of servlet engine or port number of HTTP service, click the Server tab and then click the ellipsis button next to the desired server parameter. The Edit Value dialog box appears.
  2. Type in a value, and click OK. The edited server parameter values appear in the Edit and Replay dialog box. Note that success depends upon the target server having the monitor filter installed.
Session cookies are handled according to the settings specified on the Monitor toolbar. If you chose "use the browser's session cookie for replay," then any value that was saved when the record was recorded is ignored, and the session cookie generated by the browser will be used. If you chose "use the recorded session cookie for replay," then any changes you specify in the Edit and Replay dialog box will be used. If the record header doesn't include a session cookie, then the browser won't send a cookie either. You can edit cookies either in the header or on the Cookies tab. Not all servers support this functionality.
