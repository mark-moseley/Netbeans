Working with Modules

The following table shows how to work with modules.

Create
the module project
  • Create the project using the best module project template for your programming needs.

Use
the file templates
  1. Right-click a module project in the Projects or Files window. Choose New > File/Folder.
  2. In the New File wizard, choose the best module file template for your programming needs.

Develop
the module
  1. In the Projects window or Files window, double-click the file that you would like to edit.
  2. Use the Source Editor to edit the file.
  3. Refer to the

Build
the module
  • Choose Build > Build Main Project or right-click any project and choose Build Project.

Test and debug
the module
  • Right-click a module project node and choose Install/Reload in Target Platform or Install/Reload in Development IDE to test the module.

Distribute
the module
  1. Right-click the module project and choose Create NBM. An NBM file is created.
  2. Distribute the NBM file for installation via the Update Center.

Brand and distribute
a rich-client application
  1. Right-click the module suite node in the Projects window, choose Properties and use the Application panel and Splash Screen panel to brand the application. The Application panel lets you specify a branding name for the executable, an application title for the title bar, and an icon for the Help | About dialog box. The Splash Screen panel lets you specify a splash screen and a progress bar.
  2. You can distribute the application in one or both of the following ways:
    • Create ZIP Distribution
      • Right-click the module suite node and choose Build ZIP Distribution. The application's executable and its clusters are packaged in a ZIP distribution.
      • Distribute the ZIP file.
    • Create JNLP Application
      • Right-click the module suite node and choose Build JNLP Application. The application's JNLP files are created.
      • Put the JNLP application on a host.
