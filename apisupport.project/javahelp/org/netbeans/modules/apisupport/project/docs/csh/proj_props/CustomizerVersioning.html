Module Project Properties Dialog Box: API Versioning

See Also 

You open the Module Project Properties dialog box by right-clicking a module project and choosing Properties.

The API Versioning page contains the following:

  • Coe Name Base: Displays the code name base of the module.
  • Major Release Version. Specifies the major release version in manifest.mf; may be blank or a non-negative integer.
  • Specification Version. Sets OpenIDE-Module-Specification-Version in the manifest.mf file. Specifies...
  • Append Implementation Versions Automatically. Specifies...
  • Implementation Version. Specifies...
  • Regular Module|Autoload Module|Eager Module. An autoload module is only loaded when it is needed. An eager module... A regular module...
  • Public Packages. Displays a list of all packages contained in the source directory plus any class-path-extensions. If checked, the package is included among public-packages in nbproject/project.xml.
  • Export Packages Only to Friends. Is checked if and only if there is at least one entry in the list below it. The list may include code name bases of other modules and is sorted. If not empty, packages are kept in friend-packages instead and the friends are listed in nbproject/project.xml..
  • Provided Tokens. Sets OpenIDE-Module-Provides in the manifest.mf file.
