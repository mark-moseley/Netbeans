About Module Project Templates

See Also

The IDE contains a set of standard project templates for module development. The standard distribution of the IDE contains the following module project templates:

  •  Module. An empty module with a layer.xml file and a bundle.properties file.
  •  Library Wrapper Module. A library wrapped in a module project and a bundle.properties file.
  •  Module Suite. An empty module suite.

Important Files

Module projects and library wrapper module projects have an Important Files node where the IDE stores the Ant script, layer.xml file, and other project data.

The following table lists the contents of the Important Files node and the module project template in which each is found:

Item Description Availability
XML Layer The System Filesystem registration file (layer.xml). You use this file to register new items in the System Filesystem. When you use file-level templates, the IDE automatically registers items for you. For example, when you use the Action wizard, you specify that a Java class should be invoked as a menu item, toolbar button, or keyboard shortcut, and the IDE registers your specification accordingly in thelayer.xml file. Module Project
Build Script The build script called by the IDE. This build script only contains an import statement that imports targets from nbproject/build-impl.xml. Use the build.xml to override targets from build-impl.xml or to create new targets. Both
Module Manifest

The JAR manifest (manifest.mf) with sections defining attributes for the module. By default, the manifest.mf file created for a module project specifies its version, name, layer.xml, localizing bundle, and specification version.

Both
Project Metadata

The project.xml file. IDE-generated metadata file for specifying module dependencies and classpaths. Although you can edit project.xml manually, you generally do not need to. Either a file-level template, the IDE automatically specifies module dependencies required by the item in question. Should you need to specify module dependencies yourself, use the Libraries panel in the project's Project Properties dialog box.

Both
