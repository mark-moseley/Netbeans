About Plug-in Modules and Rich-Client Applications

See Also

A NetBeans plug-in module is a Java archive (JAR) file which contains Java classes written to interact with the NetBeans APIs. A plug-in module identifies itself as a module by an entry in its MANIFEST.MF file. NetBeans plug-in modules are packaged as NBM files (.nbm extension) for non-installer distribution, usually via the Update Center wizard under the Tools menu.

NetBeans plug-in modules are written with one of two aims in mind:

  • Extending the IDE. You can very easily generate skeleton code for extending the IDE's functionality with new features. For example, you can use the skeleton code to write plug-in modules that make your favorite cutting-edge technologies available to the NetBeans IDE. Or, if you miss some functionality in the IDE, you can add it yourself, by using the skeleton code to write a plug-in module that provides the desired functionality.
  • Building a rich-client application. You can use the core of the IDE as a platform on top of which you develop standalone desktop applications. The core of the IDE is a separate product called the NetBeans Platform. By loading plug-in modules into the NetBeans Platform, you can save a lot of development time by reusing the platform's existing features such as menus, toolbars, windowing systems.

    Even though it is a separate product, there is little need to download the NetBeans Platform separately—you can develop the rich-client application in the IDE and then exclude the plug-in modules that are specific to the IDE but that are superfluous to you application.
