Add Module Dependency Dialog Box

See Also 

The Add Module Dependency dialog box lets you add the modules that your plug-in module needs. When you do so, the IDE adds entries to the plug-in module's project.xml file.

You can open the Add Module Dependency dialog box by right-clicking a module project, a library wrapper module project, or a module suite project and then choosing Properties. In the Project Properties dialog box, choose Librares and then click Add next to the Module Dependencies list.

In the NetBeans Platform Manager, you can use the following fields:

  • Filter. Search for the modules you need by typing the class that you want to use. The module dependencies list narrows, listing only those modules that can provide the class that you are typing.
  • Show Non-API Modules. If unchecked (default), the IDE excludes any module for which the current module cannot access any packages without an implementation dependency. These are known as "non-API modules". In other words, only modules with public packages are included, or friend packages where this module is a friend.

Once you have found the module you need, click OK, and then the IDE will add the associated entries in the project.xml file.
