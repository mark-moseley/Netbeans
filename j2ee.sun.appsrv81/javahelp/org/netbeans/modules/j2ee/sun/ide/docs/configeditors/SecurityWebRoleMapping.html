sun-web.xml/sun-ejb-jar.xml Visual Editor: Security Role Mapping

See Also

The sun-web.xml/sun-ejb-jar.xml visual editor lets you edit the platform-specific deployment descriptor for the Sun Java System Application Server/GlassFish. The Security Role Mapping pane enables you to specify the principals and groups assigned to each security role specified in web.xml or ejb-jar.xml. The security role nodes in the pane are determined by the security roles defined in web.xml.

A web or enterprise application will only contain the sun-web.xml or sun-ejb-jar.xml deployment descriptors if the target server is the Sun Java System Application Server/GlassFish.

You get to this pane by expanding your project's Configuration Files node in the Projects window and double-clicking sun-web.xml or sun-ejb-jar.xml and clicking the Security tab.

You can click the XML tab at the top of the visual editor to edit the file in the XML editor.

For each security role you can specify the following security role mapping properties:

  • Principals Assigned to this Role. This field specifies the names of the principals assigned to the role. Use the buttons on the right to add, edit and remove principals.
  • Groups Assigned to this Role. This field specifies the names of the groups assigned to the role. Use the buttons on the right to add, edit and remove groups.

The names of the principals and groups specified in sun-web.xml must also be defined on the Sun Java System Application Server/GlassFish. For more on working with users and groups, see Managing Users for the Sun Java System Application Server/GlassFish.

To add a security role:

  • To add a security role node to sun-web.xml, create the security role in web.xml. For steps on adding security roles in web.xml, see web.xml Visual Editor: Security Roles.
