Registering an SJS Application Server Instance

See Also

To begin developing enterprise applications you must first register an instance of the SJS Application Server with the IDE.

If you deploy your applications to a remote instance of the SJS Application Server, those libraries are available at runtime. However, during development, to compile your project you need a local instance of this server and libraries.

You can have multiple instances of the SJS Application Server registered with the IDE, but you must first specify the local folder that contains the local installation of the SJS Application Server. You only need to specify the location of the local installation once.

To register an instance of the SJS Application Server:

  1. Choose Tools  Server Manager.
  2. In the Server Manager, click Add Server to start the Add Server wizard.
    • Alternatively, right-click the Servers node in the Runtime window and choose Add Server from the contextual menu.
  3. Select the Sun Java System Application Server 8.1 from the combo box.
  4. Enter a name for the server instance. This name is used to identify the server in the IDE.
  5. Click Next.
  6. Supply the installation location of the Sun Java System Application Server 8.1 and click Next. You only need to specify the location the first time you register an instance of the server. This step in the wizard is skipped when you register additional instances of the server.
  7. Enter the location (host:port) or select one from the combo box.
  8. Specify the Domains Folder and Instance Name, if necessary.
  9. (optional) Enter the username and password.
  10. Click Finish to return to the Server Manager dialog box.
  11. Click Close.
  • If you do not enter the username and password when you register the server, when the server starts a dialog box appears asking for the username and password.

Once a server instance is registered with the IDE, its node is visible in the Runtime window under the Servers node. You can perform the following tasks by right-clicking the server instance node and choosing from the contextual menu:

  • Start and stop the SJS Application Server instance.
  • Remove the instance of the SJS Application Server.
  • Open the SJS Application Server admin console.
  • View SJS Application Server log file.
  • View the SJS Application Server properties.

To remove an instance of the SJS Application Server:

  1. Choose Tools  Server Manager.
  2. In the Server Manager, select the SJS Application Server from the list of servers in the left pane.
  3. Click Remove.
  4. Click Close.
  • Alternatively, right-click the server instance in the Runtime window and choose Remove from the contextual menu.
