Visual Editor: Message Security Bindings Dialog Box

See Also

This dialog box enables you to specify the message security bindings for a web service endpoint or port. You open this dialog box by selecting the Message Security option for a web service endpoint in the deployment descriptor visual editor and clicking Edit Bindings.

You also use this dialog box when configuring message security policy for a port for web service clients.

You use this dialog box to associate a specific provider or message protection policy with a web services endpoint or service reference.

This dialog box enables you to edit the following properties:

  • Authorization Layer. Specifies the message layer at which authentication is performed. The value must be SOAP.
  • Provider ID. Specifies the authentication provider used to satisfy application-specific message security requirements. If not specified, a default provider is used, if it is defined for the message layer. If no default provider is defined, authentication requirements defined in the message-security-binding are not enforced.
  • Message Security Entries.

    Use the buttons below the textfield to edit, remove and create new message security entries.

For more information on configuring message security, see "Using Message Security with Java WSDP" in the Java Web Services Tutorial.
