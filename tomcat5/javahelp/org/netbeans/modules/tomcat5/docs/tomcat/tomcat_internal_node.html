The  node in the Runtime window represents a physical installation of a Tomcat server. When you install the IDE, the installer automatically adds an an instance for the bundled Tomcat server. This installation cannot be removed. You can add additional Tomcat servers using the Add New Server dialog box.

Working With This Node

If you right-click this node and display the contextual menu, you see a list of menu items that enable you to work with this node. The following list describes some of these menu items and provides links to topics that explain more.

  • Start/Stop Server. Displays the Server Status dialog box. If the server is running, the Stop Server button is enabled. If the server is not running, the Start Server button is enabled.
  • Refresh. Shows the status (started or stopped) of the server.
  • Remove. Removes the server from the Server Registry. Note that you cannot remove the bundled Tomcat server.
  • View Shared Context Log. Displays the server log file as defined in the server.xml file. This item is enabled when the server is running. If no logger is defined in the context.xml file, logging for that context is performed in this shared context log.
  • Properties. Opens the Tomcat Server instance's Properties dialog box.
  • Set as Default. Tells the IDE to use this server for executing web applications. The current default server is shown in parentheses next to the Server Registry node.
