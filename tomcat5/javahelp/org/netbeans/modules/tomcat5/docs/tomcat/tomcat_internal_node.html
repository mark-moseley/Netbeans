The  node in the Runtime window represents a physical installation of a Tomcat Web Server. When you install the IDE, the installer automatically adds an an instance for the bundled Tomcat Web Server. This installation cannot be removed. You can register additional Tomcat Web Servers using the Add New Server dialog box.

Working With This Node

If you right-click this node and display the contextual menu, you see a list of menu items that enable you to work with this node. The following list describes some of these menu items and provides links to topics that explain more.

  • Start/Stop Server. Displays the Server Status dialog box. If the server is running, the Stop Server button is enabled. If the server is not running, the Start Server button is enabled.
  • Refresh. Shows the status (started or stopped) of the server.
  • Remove. Removes the server from the Servers node. Note that you cannot remove the bundled Tomcat Web Server.
  • Edit server.xml. Opens the server.xml file in the Source Editor. It is recommended to edit this file using the Tomcat Manager. Only advanced users should edit this file in the Source Editor.
  • View Server Output. Displays the server log file as defined in the server.xml file. This item is enabled when the server is running. If no logger is defined in the context.xml file, logging for that context is performed in this shared context log.
  • Properties. Opens the Tomcat Web Server instance's Properties dialog box.
