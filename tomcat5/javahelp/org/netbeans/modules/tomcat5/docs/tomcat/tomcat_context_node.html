Tomcat Context Node

See Also

The context node represents a web module context that has been deployed to the server. It appears in the Web Modules folder under the server's node in the Runtime window.

The IDE creates a web module's context node the first time you execute the web module from the IDE.

When the server receives the HTTP request, the server selects the web module for processing the request by matching the longest possible prefix of the Request URI against all of the defined context paths.

Working With This Node

If you right-click this node and display the contextual menu, you see a list of menu items that enable you to work with this node. The following list describes these menu items and provides links to topics that explain more.

  • Start. Restarts a stopped web module.
  • Stop. Disables the web module so that no one can run the module. You can use this action to prevent others from using the module while fixing or modifying the module. If someone tries to run a stopped web module, the server returns HTTP Status 503, which indicates that the application is not currently available.
  • Undeploy. Removes the web module's context from the server. The IDE will recreate the context if the web module is subsequently executed from the IDE.
