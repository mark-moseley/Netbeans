Customizing a Tomcat Server

See Also

You open the Tomcat Instance property sheet from a server's instance node in the Runtime window. A Tomcat instance node has the following properties.

Properties Section

  • Server Port. The number of the TCP/IP port that the server listens on. The URL for a web module that is deployed on the server is derived from the host's URL and the server port, such as http://localhost:8086.
  • Admin Port. The number of the TCP/IP port that the server's administrative server listens to.
  • User. The user name that the IDE uses to log into the server's manager application. The user must be associated with the manager role. The first time the IDE starts the server, such as through the Start/Stop Server menu action or by executing a web component from the IDE, the IDE adds an admin user with a randomly-generated password to the tomcat-base-path/conf/tomcat-users.xml file. The entry looks similar to the following: 
      <user username="ide" password="woiehh" roles="manager"/>
  • Password. The user's password.
  • Home Directory. The home directory for the server on your local system. This property is set when you add a Tomcat server to the Server Registry. This location is where Tomcat expects to find its own binaries and any libraries that are used by web modules but which are not part of Tomcat itself.
  • Base Directory. The base directory for the Tomcat server. You can set this property when you add a Tomcat server to the Server Registry. If you set a base directory, then that is where the Tomcat configuration files will reside. One reason for specifying a separate base directory is to allow more than one user to use the same server. If no base directory is specified, then the configuration files reside in the home directory.
  • Enable HTTP Monitor. Select this checkbox to enable the HTTP Monitor for web modules executing on the server. When enabling the monitor, you must restart the server in order for the change to take affect. The HTTP Monitor is useful for debugging web modules. If you are using the server for production purposes only, you may want to clear the checkbox to reduce its impact on the server's performance. When you clear the checkbox, the IDE removes the HTTP monitor's declaration from the server. However the monitor's libraries remain deployed to the server.
    • When you disable the HTTP Monitor, you do not need to restart the server for the change to take affect.

Debugger Section

  • Debugger Port. Specifies the port number that the server uses for debugging.
  • Classic. Select the checkbox to use the "classic" JDK 1.1 JVM for debugging. Clear the box to use HotSpot JVM.
  • Debugging Type. Choose one of the following types to specify how the IDE should connect to the server being debugged:
    • Socket
    • (Windows only) Shared memory
  • (Windows only) Name. Use this property to specify the identification string for shared memory access. To change the string, click the ellipsis button (...) to display the Name property editor. In the property editor, type a name, and click OK.
