Customizing a Tomcat Server

See Also

To customize a Tomcat Server:

  1. Choose Window  Runtime (Ctrl+5) from the main menu. The Runtime window appears.
  2. Expand the Server Registry node in the tree view to view the Tomcat Server nodes.
  3. Right-click a Tomcat Server's instance and select Properties from the contextual menu.

A Tomcat Server's instance node has two sections:

Properties Section

  • Server Port. The number of the TCP/IP port that the server listens on. The URL for a web application that is deployed on the server is derived from the host's URL and the server port, such as http://localhost:8086.
  • Admin Port. The number of the TCP/IP port that the server's administrative server listens to.
  • User. The user name that the IDE uses to log into the server's manager application. The user must be associated with the manager role. The first time the IDE starts the bundled Tomcat server, such as through the Start/Stop Server menu action or by executing a web component from the IDE, the IDE adds an admin user with a randomly-generated password to the tomcat-base-path/conf/tomcat-users.xml file. (Right-click the Tomcat Server instance node in the Runtime window and select Properties. In the Properties dialog box, the Base Directory property points to the base-dir directory.) The admin user entry in the tomcat-users.xml file looks similar to the following: 
      <user username="ide" password="woiehh" roles="manager"/>
  • Password. The user's password. See the "User" explanation above for details.
  • Home Directory. The home directory for the server on your local system. This property is set when you add a Tomcat server to the Server Registry. This location is where Tomcat expects to find its own binaries and any libraries that are used by web applications but which are not part of Tomcat itself.
  • Base Directory. The base directory for the Tomcat server. You can set this property when you add a Tomcat server to the Server Registry. If you set a base directory, then that is where the Tomcat configuration files will reside. One reason for specifying a separate base directory is to allow more than one user to use the same server. If no base directory is specified, then the configuration files reside in the home directory.
  • Enable HTTP Monitor. Select this checkbox to enable the HTTP Monitor for web applications executing on the server. When enabling the monitor, you must restart the server in order for the change to take affect. The HTTP Monitor is useful for debugging web applications. If you are using the server for production purposes only, you may want to clear the checkbox to reduce its impact on the server's performance. When you clear the checkbox, the IDE removes the HTTP monitor's declaration from the server. However the monitor's libraries remain deployed to the server.
    When you disable the HTTP Monitor, you do not need to restart the server for the change to take affect.
  • Open Context Log on Run. Select this checkbox to enable the context log to be opened while the web application is being deployed. You can watch the stack trace as it builds.

Debugger Section

  • Debugger Port. Specifies the port number that the server uses for debugging.
  • Classic. Select the checkbox to use the "classic" JDK 1.1 JVM for debugging. Clear the box to use HotSpot JVM.
  • Debugging Type. Choose one of the following types to specify how the IDE should connect to the server being debugged:
    • Socket
    • (Windows only) Shared memory
  • (Windows only) Name. Use this property to specify the identification string for shared memory access. To change the string, click the ellipsis button (...) to display the Name property editor. In the property editor, type a name, and click OK.
