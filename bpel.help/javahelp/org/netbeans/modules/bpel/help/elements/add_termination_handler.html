BPEL Designer: Adding Termination Handlers

See Also

The termination handler is used to control the termination of a scope. The termination of a scope happens if a scope or process enclosing it has faulted.

When a fault is thrown inside a scope or process, a fault handler associated with the scope or process should be run, but before that all the running activities inside the faulted scope or process should be terminated. If a faulted scope or process has any enclosed scopes, they also should be terminated. Terminating a scope means terminating activities inside it and executing the termination handler associated with the scope.

Note that the completion condition in For Each element may also trigger termination of enclosed scopes.

The termination handler is a container for the activities that will be performed in case a scope is terminated. You can add one termination handler for a scope.

To add a Termination Handler to Scope or Process elements:

  1. In the Design view right-click the Scope element and choose Add > Termination Handler.
    An empty container element appears.
  2. From the Palette, drag one or several activities that will be executed and place them inside the Termination Handler container. Configure the properties of each activity.
