Monitoring Execution of BPEL Processes in the BPEL Debugger

See Also

When a running process reaches a breakpoint, the Design view highlights the current position of the debugger and uses colors to differentiate between the states of BPEL activities. As the process advances, the colors and icons for the activities on the diagram are updated to reflect the execution progress. The following table lists the icons and colors that show the execution of BPEL activities.

Icon or Color Meaning
Green The breakpoint set for the activity is reached.
Gray The activity has never been executed.
The activity is now being executed.
The activity has been successfully completed.

You can also monitor the execution of current BPEL process instances in the BPEL Process Execution window.
