About Debugging BPEL Processes

See Also

Debugging BPEL processes follows the same main principles as debugging Java applications. Debugging BPEL processes entails setting breakpoints in the source code and executing the process step-by-step within a debugging session.

The main steps in debugging BPEL processes are the following:


Prepare to debug
  • Confirm that the Application Server has started.
  • Create test cases (for sample processes, test cases are automatically created; for new projects, you need to create at least one test case).
  • Set breakpoints in the BPEL process. If needed, you can also set fault breakpoints.
  • (Optional) Set watches to monitor the values of XPath expressions.


Start a debugging session

  • Start a debugging session.

Monitor your BPEL process
  • Run one or several test cases.
  • View execution of BPEL processes on the diagram in the Design view and in the BPEL Process Execution window.
  • View data in the BPEL Process Instances, BPEL Partner Links, Local Variables, and Watches windows.
  • Step through the code when a process is suspended at a breakpoint.

Finish a debugging session
  • Finish the debugging session.
