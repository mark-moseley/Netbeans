BPEL Designer: Creating a BPEL Diagram

See Also

This topic describes typical tasks you have to perform to design a BPEL Process diagram.


Create a
BPEL Module project
  • Create a BPEL Module project.
    You can choose either to create new projects or sample projects that contain skeletal BPEL and WSDL files.


Add files to the project

  • Add the following files to your project:
    • BPEL files. Note that the BPEL Design view supports the BPEL 2.0 language specification.
    • WSDL files. Create a new WSDL file or add an existing WSDL file.
    • XML schema files. Create a new .xsd file or add an existing .xsd resource.

Open BPEL file in the Design view
  • Open the BPEL file in the Design view.

Author a business process
  • In the Design view, author your business process by adding and configuring BPEL activities. You can also use the tips of the BPEL Validator displayed on the diagram to check the correctness of your business process.
    See Operations with Elements in the Design View and About BPEL Elements for details.

Verify the BPEL source code
  • (Optional) In the Source view, check the resulting code if necessary.

Deploy and perform test runs of your process
  • See Deploying and Performing Test Runs of BPEL Processes for details about deploying and performing test runs of BPEL processes.
  • If your process is asynchronous and you need it to manage multiple, stateful, long-running conversations between the process and its partner services, you can define and add one or more correlation sets.
